This example page shows a common pattern for preparing multimodal training-data releases with Vane Data.

## Goal

Prepare multimodal data by selecting raw records, running a batch captioning or labeling UDF, filtering by quality, deduplicating by content hash, generating embeddings, and writing a versioned Parquet release.

## Run local samples

The repository also includes text-focused sample scripts you can run locally to inspect the same relation patterns without external storage or models:

```bash
python examples/common_crawl.py --source sample --limit 5
python examples/minhash_dedupe.py --source sample --limit 10
python examples/llms_red_pajamas.py --source sample --limit 6
```

## Website code shape

The website shows the shortened pipeline shape. The full example below expands the `CaptionAndScore` UDF and schema.

```python
import vane
con = vane.connect()
raw = con.sql("""
select id, uri, media_type, text, metadata, content_hash
from read_parquet('s3://training-corpus/manifest/*.parquet')
where split = 'train'
""")
labeled = raw.map_batches(
    CaptionAndScore, schema=release_schema,
    execution_backend="ray_actor", gpus=1, batch_size=64,
)
labeled.to_table("labeled")
release = con.sql("""
select * exclude rn
from (
  select *, row_number() over (
    partition by content_hash order by quality_score desc
  ) as rn
  from labeled
  where quality_score >= 0.8
)
where rn = 1
""").embed_text(
    "caption",
    provider="transformers",
    execution_backend="ray_actor",
)
release.write_parquet("s3://dataset-releases/mm-v42/")
```

## Full UDF contract

`CaptionAndScore` is your batch UDF for decoding media, running GPU captioning or labeling, and returning stable release columns. Keep the schema narrow and explicit so SQL can own the release filtering and deduplication steps.

```python
import pyarrow as pa

release_schema = {
    "id": "VARCHAR",
    "uri": "VARCHAR",
    "media_type": "VARCHAR",
    "content_hash": "VARCHAR",
    "caption": "VARCHAR",
    "quality_score": "DOUBLE",
    "label": "VARCHAR",
    "metadata": "VARCHAR",
}


class CaptionAndScore:
    def __init__(self) -> None:
        self.model = load_caption_or_label_model()

    def __call__(self, batch: pa.Table) -> pa.Table:
        media = decode_media(batch.column("uri"), batch.column("media_type"))
        predictions = self.model.predict(media)

        return pa.table({
            "id": batch.column("id"),
            "uri": batch.column("uri"),
            "media_type": batch.column("media_type"),
            "content_hash": batch.column("content_hash"),
            "caption": [item.caption for item in predictions],
            "quality_score": [item.quality_score for item in predictions],
            "label": [item.label for item in predictions],
            "metadata": batch.column("metadata"),
        })
```

The helper functions above are deliberately placeholders. In a production pipeline, define them in an importable module available to every Ray worker, load model state in `__init__`, and return Arrow-compatible values from `__call__`.

## Text-only variant

For text training or retrieval data, the same relation shape applies: select raw records, run a batch UDF, embed the output column, and write the release. The chunking UDF below returns all columns needed by later stages.

```python
def chunk_rows(batch: pa.Table) -> pa.Table:
    ids = batch.column("id").to_pylist()
    texts = batch.column("text").to_pylist()
    source_uris = batch.column("source_uri").to_pylist()

    out_id = []
    out_chunk_id = []
    out_text = []
    out_source_uri = []
    for row_id, text, source_uri in zip(ids, texts, source_uris, strict=False):
        chunks = [text[i : i + 512] for i in range(0, len(text), 512)]
        for chunk_id, chunk in enumerate(chunks):
            out_id.append(row_id)
            out_chunk_id.append(chunk_id)
            out_text.append(chunk)
            out_source_uri.append(source_uri)

    return pa.table({
        "id": out_id,
        "chunk_id": out_chunk_id,
        "chunk_text": out_text,
        "source_uri": out_source_uri,
    })

docs = con.sql("""
    select id, text, source_uri
    from read_parquet('data/raw_text/*.parquet')
    where text is not null and length(text) > 80
""")

chunks = docs.map_batches(
    chunk_rows,
    schema={
        "id": "BIGINT",
        "chunk_id": "BIGINT",
        "chunk_text": "VARCHAR",
        "source_uri": "VARCHAR",
    },
    batch_size=512,
    execution_backend="subprocess_task",
)
```

## Embeddings

`rel.embed_text(...)` returns a relation containing the embedding output column. If the final dataset needs the original chunk columns as well, explicitly combine the embedding column with the chunk relation after validating row counts, or use a custom UDF that returns the complete output schema.

```python
embedding_only = chunks.embed_text(
    "chunk_text",
    provider="transformers",
    model="sentence-transformers/all-MiniLM-L6-v2",
    output_column="embedding",
    execution_backend="subprocess_actor",
)

chunk_table = chunks.to_arrow_table()
embedding_table = embedding_only.to_arrow_table()
embedded = con.from_arrow(chunk_table.append_column("embedding", embedding_table["embedding"]))

embedded.write_parquet("output/training_chunks/")
```

For large datasets, prefer an implementation pattern where joins or combine steps are explicit and validated, or implement embedding as a custom batch UDF that returns all required columns.

## Scale-out

Ray is already the default runner. After the bounded local path is correct, move the expensive UDF or AI helper stage from an explicit subprocess backend to Ray scheduling:

```python
embedding_only = chunks.embed_text(
    "chunk_text",
    provider="transformers",
    execution_backend="ray_actor",
)
```

Every Ray worker must be able to import the UDF code and access the same storage, model files, and Python dependencies.

## Recommended output columns

- `id`
- `uri`
- `media_type`
- `caption`
- `quality_score`
- `label`
- `content_hash`
- `embedding`
- `metadata`

## Scope notes

Vane Data does not provide a complete training-data quality framework. It provides the SQL, UDF, AI helper, and execution tools used to build that framework in your own pipeline.
