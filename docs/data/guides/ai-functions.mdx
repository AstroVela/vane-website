This guide completes one reviewable document-analysis task. It uses SQL `ai_prompt` and `ai_embed` first, shows Python Expression as an alternate entry point to the same row-preserving model, and moves to Relation API only for capabilities that require it.

Use the [AI Function API Reference](/docs/data/reference/ai-api) for exact signatures and options, and [AI Function Concepts](/docs/data/concepts/ai-functions) for the reasons behind the Expression and Relation models.

**Prerequisites**

- Install the OpenAI client in every environment that may execute the relation, including every worker environment: `python -m pip install openai`.
- Set `OPENAI_API_KEY` in every worker's environment. Do not put credentials in SQL text or options.
- Provide Parquet files at `data/documents/*.parquet` with `document_id`, `document_type`, `source_uri`, and `text` columns that can be cast to `VARCHAR`. In the selected slice, IDs must be non-null and unique, and source URIs and text must be non-null.
- Ensure the output parent directory exists before running, for example with `mkdir -p output`.

## 1. Prepare a Reviewable Source

Start with a stable business ID, the source text sent to the model, and enough metadata to reproduce and review an output:

```python
import vane

con = vane.connect()
source = con.sql(
    """
    SELECT
        CAST(document_id AS VARCHAR) AS document_id,
        CAST(document_type AS VARCHAR) AS document_type,
        CAST(source_uri AS VARCHAR) AS source_uri,
        CAST(text AS VARCHAR) AS source_text
    FROM read_parquet('data/documents/*.parquet')
    """
)
source.to_table("ai_review_source")
```

`document_id` is the review and write key. `source_uri`, `document_type`, and `source_text` explain where each model result came from and what the provider saw.

## 2. Complete the Task with SQL Expression

### Select a Candidate Slice

Use ordinary SQL to filter before any provider call. Keep the first run small, deterministic, and representative:

```python
candidates = con.sql(
    """
    SELECT
        'ai-review-2026-07-11-001'::VARCHAR AS run_id,
        document_id,
        document_type,
        source_uri,
        source_text,
        (
            'Summarize the business issue and required follow-up in one sentence: '
            || source_text
        )::VARCHAR AS review_prompt
    FROM ai_review_source
    WHERE source_text IS NOT NULL
      AND trim(source_text) <> ''
      AND document_type IN ('invoice', 'claim', 'memo')
    ORDER BY document_id
    LIMIT 100
    """
)
candidates.to_table("ai_review_candidates")

expected_ids = [row[0] for row in con.sql(
    "SELECT document_id FROM ai_review_candidates ORDER BY document_id"
).fetchall()]
expected_row_count = len(expected_ids)

if expected_row_count == 0:
    raise ValueError("AI review candidate slice is empty")
if len(set(expected_ids)) != expected_row_count:
    raise ValueError("document_id must be unique in the candidate slice")
```

The stable ordering makes this validation run reproducible. The ID and source fields remain in every later projection; model output order is never used as an identity contract.

### Generate Review Notes First

Set `OPENAI_API_KEY` in the environment of every worker before executing the relation. Never place a key, token, password, or secret in SQL options.

```python
prompted = con.sql(
    """
    SELECT
        run_id,
        document_id,
        document_type,
        source_uri,
        source_text,
        review_prompt,
        ai_prompt(
            review_prompt,
            struct_pack(
                provider := 'openai',
                model := 'gpt-4o-mini',
                concurrency := 2,
                max_api_concurrency := 4,
                batch_size := 8,
                max_retries := 2,
                on_error := 'log',
                system_message := 'Return one concise factual review note.',
                temperature := 0.0
            )
        ) AS review_note
    FROM ai_review_candidates
    ORDER BY document_id
    """
)
prompted.to_table("ai_review_prompted")
```

The second argument is one foldable constant `STRUCT`. Provider, model, execution, and request settings cannot vary by row. With `on_error := 'log'`, a response that still fails after retries remains a null `review_note` for the validation step instead of silently disappearing.

### Add Embeddings When Retrieval Is Part of the Task

This guide's task produces a review artifact for both audit and retrieval, so the workflow below assumes the `embedding` column is present. Add one more SQL Expression column while retaining the source and prompt output:

```python
reviewed = con.sql(
    """
    SELECT
        run_id,
        document_id,
        document_type,
        source_uri,
        source_text,
        review_prompt,
        review_note,
        ai_embed(
            source_text,
            struct_pack(
                provider := 'openai',
                model := 'text-embedding-3-small',
                dimensions := 512,
                normalize := true,
                concurrency := 2,
                batch_size := 32,
                max_retries := 2,
                on_error := 'raise'
            )
        ) AS embedding
    FROM ai_review_prompted
    ORDER BY document_id
    """
)
reviewed.to_table("ai_review_model_output")
```

Both calls run inside `SELECT` projections and can be nested in ordinary projection expressions. Each adds one typed, row-preserving output column; predicates, grouping, joins, short-circuit composition, and row-varying configuration stay outside the AI expression.

### Validate and Write the Review Artifact

Turn tolerant prompt failures into an explicit review status, then check cardinality, IDs, null/error output, and the exact schema before writing:

```python
reviewable = con.sql(
    """
    SELECT
        run_id,
        document_id,
        document_type,
        source_uri,
        source_text,
        review_prompt,
        review_note,
        embedding,
        CASE
            WHEN review_note IS NULL OR trim(review_note) = ''
                THEN 'prompt_error'
            WHEN embedding IS NULL OR array_length(embedding) <> 512
                THEN 'embedding_error'
            ELSE 'ready'
        END::VARCHAR AS ai_status
    FROM ai_review_model_output
    ORDER BY document_id
    """
)
reviewable.to_table("ai_review_result")

validation = con.sql(
    """
    SELECT
        count(*) AS row_count,
        count(DISTINCT document_id) AS distinct_id_count,
        count(*) FILTER (WHERE document_id IS NULL) AS null_id_count,
        count(*) FILTER (
            WHERE source_uri IS NULL OR source_text IS NULL
        ) AS null_source_count,
        count(*) FILTER (WHERE ai_status <> 'ready') AS error_output_count
    FROM ai_review_result
    """
).fetchone()
(
    actual_row_count,
    distinct_id_count,
    null_id_count,
    null_source_count,
    error_output_count,
) = validation

assert actual_row_count == expected_row_count
assert distinct_id_count == expected_row_count
assert null_id_count == 0
assert null_source_count == 0

actual_ids = [row[0] for row in con.sql(
    "SELECT document_id FROM ai_review_result ORDER BY document_id"
).fetchall()]
assert actual_ids == expected_ids

expected_columns = [
    "run_id",
    "document_id",
    "document_type",
    "source_uri",
    "source_text",
    "review_prompt",
    "review_note",
    "embedding",
    "ai_status",
]
expected_types = [
    "VARCHAR",
    "VARCHAR",
    "VARCHAR",
    "VARCHAR",
    "VARCHAR",
    "VARCHAR",
    "VARCHAR",
    "FLOAT[512]",
    "VARCHAR",
]
assert list(reviewable.columns) == expected_columns
assert [str(dtype) for dtype in reviewable.types] == expected_types

print({"error_output_count": error_output_count})
reviewable.write_parquet("output/ai-review-2026-07-11-001.parquet")
```

The write keeps failed rows beside their IDs, source, prompt, and status for review. Require `error_output_count == 0` only when the next stage accepts clean results exclusively. Use a new `run_id` and output path for a genuinely new provider run; for a durable table, merge or replace by stable keys instead of blindly appending retries.

## 3. Use Python Expression as an Alternate

Use Python Expression when typed provider/configuration construction belongs in Python. It has the same one-column, row-preserving projection semantics as SQL Expression; it is not the path to structured, multimodal, or table-shaped output.

```python
python_prompt_provider_options = vane.ai.OpenAIProviderOptions(
    concurrency=2,
    max_api_concurrency=4,
)
python_embedding_provider_options = vane.ai.OpenAIProviderOptions(
    concurrency=2,
)
python_prompt_options = vane.ai.OpenAIPromptOptions(
    temperature=0,
    on_error="log",
)
python_embedding_options = vane.ai.OpenAIEmbeddingOptions(
    on_error="raise",
)

python_prompted = candidates.select(
    vane.col("run_id"),
    vane.col("document_id"),
    vane.col("document_type"),
    vane.col("source_uri"),
    vane.col("source_text"),
    vane.col("review_prompt"),
    vane.ai.prompt(
        vane.col("review_prompt"),
        provider="openai",
        model="gpt-4o-mini",
        provider_options=python_prompt_provider_options,
        prompt_options=python_prompt_options,
        system_message="Return one concise factual review note.",
    ).alias("review_note"),
)

python_reviewed = python_prompted.select(
    vane.col("run_id"),
    vane.col("document_id"),
    vane.col("document_type"),
    vane.col("source_uri"),
    vane.col("source_text"),
    vane.col("review_prompt"),
    vane.col("review_note"),
    vane.ai.embed(
        vane.col("source_text"),
        provider="openai",
        model="text-embedding-3-small",
        provider_options=python_embedding_provider_options,
        embedding_options=python_embedding_options,
        dimensions=512,
        normalize=True,
    ).alias("embedding"),
)
python_reviewed.to_table("ai_review_python_model_output")
```

Apply the same row-count, ordered-ID, null/error, schema, and run-specific write checks used for `ai_review_model_output`. Keep every AI Expression directly inside `rel.select(...)` with an alias; do not use it in filters or manually recombine its output with source rows.

## 4. Use Relation API Only for Specialized Capabilities

Relation helpers are Python table transformations. Vane does not provide a direct SQL Relation or table-function API for them. Use them only when the task requires capabilities that SQL or Python Expression does not expose.

### Structured or Multimodal Prompting

Define the response model and import before calling `rel.prompt`. The same helper accepts `image_columns` for provider/model combinations that support image input:

```python
from pydantic import BaseModel


class ReviewDecision(BaseModel):
    label: str
    reason: str


multimodal_candidates = con.sql(
    """
    SELECT
        CAST(document_id AS VARCHAR) AS document_id,
        CAST(review_prompt AS VARCHAR) AS review_prompt,
        image
    FROM read_parquet('data/review-items/*.parquet')
    WHERE review_prompt IS NOT NULL
    ORDER BY document_id
    LIMIT 20
    """
)

decision_only = multimodal_candidates.prompt(
    "review_prompt",
    image_columns=["image"],
    provider="openai",
    model="gpt-4o-mini",
    provider_options=vane.ai.OpenAIProviderOptions(
        max_api_concurrency=4,
    ),
    prompt_options=vane.ai.OpenAIPromptOptions(
        temperature=0,
        on_error="log",
    ),
    return_format=ReviewDecision,
    output_column="decision_json",
    execution_backend="subprocess_task",
)
```

Validate every non-null `decision_json` against `ReviewDecision` before publishing it. Use text-only input by omitting `image_columns`; keep `return_format` only when the selected provider supports structured responses.

### Built-In Classification

`rel.classify_text` is the built-in zero-shot classification path:

```python
label_only = candidates.classify_text(
    "source_text",
    labels=["billing", "delivery", "policy", "other"],
    provider="transformers",
    model="facebook/bart-large-mnli",
    output_column="review_label",
    batch_size=16,
    execution_backend="subprocess_task",
)
```

Validate non-null labels against the declared set and sample every label group before scaling.

### Built-In Long-Text Chunking

`rel.embed_text` owns built-in character chunking through `max_chunk_chars`:

```python
chunk_embedding_only = candidates.embed_text(
    "source_text",
    provider="transformers",
    model="sentence-transformers/all-MiniLM-L6-v2",
    max_chunk_chars=2000,
    chunk_overlap_chars=200,
    output_column="chunked_embedding",
    execution_backend="subprocess_task",
)
```

Use SQL `ai_embed` or `vane.ai.embed` when each existing row simply needs one embedding column. Use this Relation helper when the helper must split and combine long text itself; use the [Embeddings at Scale Guide](/docs/data/guides/embeddings-at-scale) when chunks need their own stable IDs and rows.

### Select an Explicit Relation Backend and Actor Pool

Relation methods can select `execution_backend` explicitly. For a Ray-backed vLLM actor pool, provider `concurrency` maps to actor count while `gpus_per_actor` requests the Ray GPU resource for each actor:

```python
vane.configure(runner="ray")

gpu_response_only = candidates.prompt(
    "review_prompt",
    provider="vllm",
    model="Qwen/Qwen3-1.7B",
    provider_options=vane.ai.VLLMProviderOptions(
        concurrency=2,
        gpus_per_actor=1,
    ),
    prompt_options=vane.ai.VLLMPromptOptions(
        max_tokens=128,
        temperature=0,
        on_error="log",
    ),
    output_column="gpu_review_note",
    execution_backend="ray_actor",
)
```

The `_only` names are deliberate: Relation helpers may return only their configured output columns rather than automatically retaining every source field. Plan source preservation before the call. Use the built-in helper as an isolated output-only stage only when downstream work does not need source keys. When it does, use a custom row-preserving batch UDF that returns the stable IDs and outputs in one declared schema. Never reconnect results by row position.

## 5. Operate Providers and Writes Safely

- **Provider and model:** Record the exact provider, model, prompt/run version, and output path. Hosted providers add network, quota, latency, and cost concerns; local providers add dependency, model-file, memory, and startup concerns.
- **Credentials:** Provider descriptors are serialized and instantiated lazily on workers. Put credentials in each worker environment, not in SQL text or serialized options, and limit worker network access to the required endpoint.
- **Execution controls:** SQL and Python Expression let the active runner choose a local subprocess actor or Ray actor. Relation methods can choose an explicit backend. Provider `concurrency` controls actor count; `max_api_concurrency` limits calls inside an actor; batch size affects throughput, memory, and failure scope.
- **Retries and errors:** Validate with `on_error="raise"` when failures should stop the sample. Tolerant modes can turn prompt failures into nulls and, when the dimension is known, embedding failures into zero vectors, so check the output contract you selected. Wrapper and provider-SDK retries can both repeat a call.
- **External effects:** Provider requests are not transactional or exactly-once. A query retry can incur another request and charge, and rollback cannot undo a completed call. Preserve `(run_id, document_id)`, write reviewable run artifacts, and make downstream merges idempotent.
