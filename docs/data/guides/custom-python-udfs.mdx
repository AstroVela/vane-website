title: Custom Python UDFs
---

This guide completes one real task: normalize document text into a typed review column, keep every result traceable to a stable document ID and source URI, validate the output contract, write Parquet, and clean up the registered SQL alias.

Use the [UDF API Reference](/docs/data/reference/udf-api) for exact signatures and restrictions. Use [UDF Concepts](/docs/data/concepts/udfs) for the reasons behind the SQL-first Expression model and the Relation escape hatch.

## Task Goal and Prerequisites

The primary workflow runs locally against inline data. It requires a working Vane installation, but the task itself needs no Ray cluster, GPU, external files, credentials, or PyArrow code. It writes `output/document-review.parquet`.

The source contract has three explicitly typed columns:

| Column | Type | Purpose |
| --- | --- | --- |
| `document_id` | `VARCHAR` | Stable business ID used for row-count and identity checks. |
| `source_uri` | `VARCHAR` | Traceable source location retained in every result. |
| `text` | `VARCHAR` | Nullable input to normalize. |

For numeric UDFs, make the same type boundary explicit: use `1.25::DOUBLE` with `parameters=["DOUBLE"]` and `return_dtype="DOUBLE"`. A bare decimal literal is normally `DECIMAL`, which should not be silently treated as the same contract as `DOUBLE`.

## Primary Workflow: SQL Expression (Recommended)

Define a raw Python scalar callable first—there is no `vane.func` decorator in the primary path. Then create the connection and source, attach a typed alias to that connection, and invoke the alias only in a SQL `SELECT` projection.

```python
import vane

import re
from pathlib import Path


def normalize_review_text(text):
    if text is None:
        return None
    return re.sub(r"\s+", " ", text).strip().lower()


con = vane.connect()
source = con.sql(
    """
    SELECT
        document_id::VARCHAR AS document_id,
        source_uri::VARCHAR AS source_uri,
        text::VARCHAR AS text
    FROM (VALUES
        ('doc-001', 's3://sample/doc-001.txt', ' Missing   signature '),
        ('doc-002', 's3://sample/doc-002.txt', NULL),
        ('doc-003', 's3://sample/doc-003.txt', ' APPROVED ')
    ) AS source_rows(document_id, source_uri, text)
    ORDER BY document_id
    """
)
source.to_table("udf_source_documents")

vane.attach_function(
    normalize_review_text,
    alias="normalize_review_text_sql",
    connection=con,
    parameters=["VARCHAR"],
    return_dtype="VARCHAR",
)

try:
    projected = con.sql(
        """
        SELECT
            document_id,
            source_uri,
            text,
            normalize_review_text_sql(text) AS review_text
        FROM udf_source_documents
        ORDER BY document_id
        """
    )
    projected.to_table("udf_reviewed_documents")
    result = con.table("udf_reviewed_documents")

    expected_ids = [
        row[0]
        for row in con.sql(
            "SELECT document_id FROM udf_source_documents ORDER BY document_id"
        ).fetchall()
    ]
    rows = result.order("document_id").fetchall()
    actual_ids = [row[0] for row in rows]

    assert list(result.columns) == [
        "document_id",
        "source_uri",
        "text",
        "review_text",
    ]
    assert [str(dtype) for dtype in result.types] == [
        "VARCHAR",
        "VARCHAR",
        "VARCHAR",
        "VARCHAR",
    ]
    assert len(rows) == len(expected_ids)
    assert actual_ids == expected_ids
    assert len(set(actual_ids)) == len(expected_ids)

    source_nulls = con.sql(
        "SELECT count(*) FROM udf_source_documents WHERE text IS NULL"
    ).fetchone()[0]
    result_nulls = con.sql(
        "SELECT count(*) FROM udf_reviewed_documents WHERE review_text IS NULL"
    ).fetchone()[0]
    assert result_nulls == source_nulls
    assert rows[0][3] == "missing signature"
    assert rows[2][3] == "approved"

    output_dir = Path("output")
    output_dir.mkdir(parents=True, exist_ok=True)
    result.write_parquet(
        str(output_dir / "document-review.parquet"),
        overwrite=True,
    )
finally:
    try:
        vane.detach_function("normalize_review_text_sql", connection=con)
    finally:
        con.close()
```

The alias is connection-scoped, row-preserving in v1, and valid only in projection positions. Filter and join the source with ordinary SQL before this projection; do not put the alias in `WHERE`, `GROUP BY`, `HAVING`, join conditions, aggregate arguments, or short-circuit-sensitive expressions. The authoritative placement and registration rules are in the [UDF API Reference](/docs/data/reference/udf-api).

Materializing `udf_reviewed_documents` makes validation and the final write operate on one completed UDF result rather than rerunning the callable for every check. The assertions verify schema, exact SQL types, row count, distinct ordered IDs, null propagation, and representative values before any output is accepted. `finally` removes the alias from the same connection even when projection, validation, or writing fails.

SQL is the default because relational selection stays visible while one typed Python result composes beside the original columns. See [UDF Concepts](/docs/data/concepts/udfs) for that design rationale.

## Alternate Workflow: Python Expression

Use Python construction when expressions are assembled dynamically, reused as Python objects, or composed in a notebook or library. This is an alternate call form for the same typed projection model, not a different output model.

### Scalar Python Expression with `vane.func`

The following task produces the same four-column, row-preserving projection without registering a SQL alias:

```python
import vane

import re


@vane.func(return_dtype="VARCHAR")
def normalize_review_text(text):
    if text is None:
        return None
    return re.sub(r"\s+", " ", text).strip().lower()


con = vane.connect()
source = con.sql(
    """
    SELECT
        document_id::VARCHAR AS document_id,
        source_uri::VARCHAR AS source_uri,
        text::VARCHAR AS text
    FROM (VALUES
        ('doc-001', 's3://sample/doc-001.txt', ' Missing   signature '),
        ('doc-002', 's3://sample/doc-002.txt', NULL)
    ) AS source_rows(document_id, source_uri, text)
    """
)

result = source.select(
    vane.col("document_id"),
    vane.col("source_uri"),
    vane.col("text"),
    normalize_review_text(vane.col("text")).alias("review_text"),
)

assert list(result.columns) == [
    "document_id",
    "source_uri",
    "text",
    "review_text",
]
assert len(result.fetchall()) == 2
con.close()
```

### Arrow Batch Python Expression with `vane.func.batch`

Use a batch Expression when batching helps but the contract is still one typed output column beside the input rows:

```python
import vane

import re
import pyarrow as pa


def normalize_review_batch(table):
    values = table.column("text").to_pylist()
    normalized = [
        None if value is None else re.sub(r"\s+", " ", value).strip().lower()
        for value in values
    ]
    return pa.table({"review_text": pa.array(normalized, type=pa.string())})


con = vane.connect()
source = con.sql(
    """
    SELECT
        document_id::VARCHAR AS document_id,
        source_uri::VARCHAR AS source_uri,
        text::VARCHAR AS text
    FROM (VALUES
        ('doc-001', 's3://sample/doc-001.txt', ' Missing   signature '),
        ('doc-002', 's3://sample/doc-002.txt', NULL)
    ) AS source_rows(document_id, source_uri, text)
    """
)

review_text = vane.func.batch(
    normalize_review_batch,
    inputs={"text": vane.col("text")},
    schema={"review_text": "VARCHAR"},
    row_preserving=True,
).alias("review_text")

result = source.select(
    vane.col("document_id"),
    vane.col("source_uri"),
    vane.col("text"),
    review_text,
)

assert len(result.fetchall()) == 2
con.close()
```

`row_preserving=True` checks that the batch output has one row per input row and permits normal columns beside it. `row_preserving=False` is the explicit Python batch Expression exception: it may change cardinality, but the batch result must be the sole top-level projection, and it cannot be registered as a SQL alias in v1. See the [UDF API Reference](/docs/data/reference/udf-api) for the exact batch and placement contracts.

## Stateful `vane.cls`: A Narrow Contract

Use `vane.cls` only when mutable state must live in one actor for one query. In v1, `actor_number=1` is required; the state is query-scoped, actor-local, ephemeral, not checkpointed, and not exactly-once. This memoizer keeps an in-query cache while still returning one deterministic typed column:

```python
import vane

import re


@vane.cls(actor_number=1, return_dtype="VARCHAR")
class QueryMemoizer:
    def __init__(self):
        self.cache = {}

    def __call__(self, text):
        if text not in self.cache:
            self.cache[text] = re.sub(r"\s+", " ", text).strip().lower()
        return self.cache[text]


memoizer = QueryMemoizer()
con = vane.connect()
source = con.sql(
    """
    SELECT
        document_id::VARCHAR AS document_id,
        text::VARCHAR AS text
    FROM (VALUES
        ('doc-001', ' Repeat   me '),
        ('doc-002', ' Repeat   me ')
    ) AS source_rows(document_id, text)
    """
)

result = source.select(
    vane.col("document_id"),
    memoizer(vane.col("text")).alias("review_text"),
)

assert len(result.fetchall()) == 2
con.close()
```

Actor loss fails the query, and distributed execution does not define a stable global row-evaluation order. Do not use this contract as durable, global, keyed, or transactional state. The exact restrictions belong to the [UDF API Reference](/docs/data/reference/udf-api); the reasons and failure model are explained in [UDF Concepts](/docs/data/concepts/udfs). If the need is only to reuse an expensive model or client and multiple independent actors are acceptable, use the actor-backed Relation pattern below.

## Relation API for Table-Shaped Capabilities

Move to Relation API only when the required capability is table-shaped or needs an explicit Relation executor. Vane does not expose a direct SQL table-function form for these UDFs, so these stages are constructed in Python. Each example below names the capability that requires Relation API.

### Capability: Arrow Multi-Column or Whole-Table Output — `rel.map_batches`

`rel.map_batches` is required here because one call returns stable source fields plus several new columns. Relation output contains only the declared schema, so pass-through fields must be returned explicitly.

```python
import vane

import re
import duckdb
import pyarrow as pa


def prepare_review_batch(table):
    document_ids = table.column("document_id").to_pylist()
    source_uris = table.column("source_uri").to_pylist()
    texts = table.column("text").to_pylist()
    review_text = [
        None if text is None else re.sub(r"\s+", " ", text).strip().lower()
        for text in texts
    ]
    text_length = [None if text is None else len(text) for text in review_text]
    status = ["missing_text" if text is None else "ok" for text in review_text]
    return pa.table(
        {
            "document_id": pa.array(document_ids, type=pa.string()),
            "source_uri": pa.array(source_uris, type=pa.string()),
            "review_text": pa.array(review_text, type=pa.string()),
            "text_length": pa.array(text_length, type=pa.int64()),
            "status": pa.array(status, type=pa.string()),
        }
    )


con = vane.connect()
source = con.sql(
    """
    SELECT
        document_id::VARCHAR AS document_id,
        source_uri::VARCHAR AS source_uri,
        text::VARCHAR AS text
    FROM (VALUES
        ('doc-001', 's3://sample/doc-001.txt', ' Missing   signature '),
        ('doc-002', 's3://sample/doc-002.txt', NULL)
    ) AS source_rows(document_id, source_uri, text)
    """
)
review_schema = {
    "document_id": duckdb.sqltypes.VARCHAR,
    "source_uri": duckdb.sqltypes.VARCHAR,
    "review_text": duckdb.sqltypes.VARCHAR,
    "text_length": duckdb.sqltypes.BIGINT,
    "status": duckdb.sqltypes.VARCHAR,
}

result = source.map_batches(
    prepare_review_batch,
    schema=review_schema,
    batch_size=2,
    execution_backend="subprocess_task",
)

assert list(result.columns) == list(review_schema)
assert len(result.fetchall()) == 2
con.close()
```

### Capability: Explicit Zero, One, or Many Rows — `rel.flat_map`

`rel.flat_map` is the explicit cardinality boundary for filtering or expansion. This task emits zero tokens for null text and many tokens for non-null text while carrying the stable ID and source URI into every emitted row.

```python
import vane

import duckdb


def split_document(row):
    text = row["text"]
    if text is None:
        return
    for token_index, token in enumerate(text.split()):
        yield {
            "document_id": row["document_id"],
            "source_uri": row["source_uri"],
            "token_index": token_index,
            "token": token.lower(),
        }


con = vane.connect()
source = con.sql(
    """
    SELECT
        document_id::VARCHAR AS document_id,
        source_uri::VARCHAR AS source_uri,
        text::VARCHAR AS text
    FROM (VALUES
        ('doc-001', 's3://sample/doc-001.txt', 'Alpha Beta'),
        ('doc-002', 's3://sample/doc-002.txt', NULL)
    ) AS source_rows(document_id, source_uri, text)
    """
)
token_schema = {
    "document_id": duckdb.sqltypes.VARCHAR,
    "source_uri": duckdb.sqltypes.VARCHAR,
    "token_index": duckdb.sqltypes.INTEGER,
    "token": duckdb.sqltypes.VARCHAR,
}

tokens = source.flat_map(
    split_document,
    schema=token_schema,
    execution_backend="subprocess_task",
)

assert tokens.order("document_id, token_index").fetchall() == [
    ("doc-001", "s3://sample/doc-001.txt", 0, "alpha"),
    ("doc-001", "s3://sample/doc-001.txt", 1, "beta"),
]
con.close()
```

### Capability: Row-Wise Scalar Output with an Explicit Relation Executor — `rel.map`

Current `rel.map` calls the scalar function with all input columns as positional values and returns the original columns plus a result column named `value`. It is not the stale pandas DataFrame batch-mapping API. Use it when the standardized pass-through-plus-`value` contract or explicit Relation backend controls are the capability you need.

```python
import vane

import duckdb


def build_review_key(document_id, source_uri, text):
    text_size = len(text or "")
    return f"{document_id}:{text_size}"


con = vane.connect()
source = con.sql(
    """
    SELECT
        document_id::VARCHAR AS document_id,
        source_uri::VARCHAR AS source_uri,
        text::VARCHAR AS text
    FROM (VALUES
        ('doc-001', 's3://sample/doc-001.txt', 'Alpha'),
        ('doc-002', 's3://sample/doc-002.txt', NULL)
    ) AS source_rows(document_id, source_uri, text)
    """
)

mapped = source.map(
    build_review_key,
    return_type=duckdb.sqltypes.VARCHAR,
    execution_backend="subprocess_task",
)
result = mapped.project(
    "document_id, source_uri, text, value AS review_key"
)

assert result.order("document_id").fetchall() == [
    ("doc-001", "s3://sample/doc-001.txt", "Alpha", "doc-001:5"),
    ("doc-002", "s3://sample/doc-002.txt", None, None),
]
con.close()
```

### Capability: Model or Client Reuse Across Batches — Actor-Backed Relation Classes

Pass a zero-argument callable class—not an already-created instance—when setup should be reused by one or more Relation actors. Each actor constructs and reuses its own instance; multiple actors provide independent reuse and parallelism, not shared or keyed state.

```python
import vane

import re
import duckdb
import pyarrow as pa


class ReviewRuleSet:
    def __init__(self):
        self.risk_pattern = re.compile(r"missing|expired", re.IGNORECASE)

    def __call__(self, table):
        document_ids = table.column("document_id").to_pylist()
        source_uris = table.column("source_uri").to_pylist()
        texts = table.column("text").to_pylist()
        needs_review = [
            bool(self.risk_pattern.search(text or ""))
            for text in texts
        ]
        return pa.table(
            {
                "document_id": pa.array(document_ids, type=pa.string()),
                "source_uri": pa.array(source_uris, type=pa.string()),
                "needs_review": pa.array(needs_review, type=pa.bool_()),
            }
        )


con = vane.connect()
source = con.sql(
    """
    SELECT
        document_id::VARCHAR AS document_id,
        source_uri::VARCHAR AS source_uri,
        text::VARCHAR AS text
    FROM (VALUES
        ('doc-001', 's3://sample/doc-001.txt', 'Missing signature'),
        ('doc-002', 's3://sample/doc-002.txt', 'Approved')
    ) AS source_rows(document_id, source_uri, text)
    """
)
review_schema = {
    "document_id": duckdb.sqltypes.VARCHAR,
    "source_uri": duckdb.sqltypes.VARCHAR,
    "needs_review": duckdb.sqltypes.BOOLEAN,
}

result = source.map_batches(
    ReviewRuleSet,
    schema=review_schema,
    batch_size=1,
    execution_backend="subprocess_actor",
    actor_number=2,
    gpus=0.0,
)

assert result.order("document_id").fetchall() == [
    ("doc-001", "s3://sample/doc-001.txt", True),
    ("doc-002", "s3://sample/doc-002.txt", False),
]
con.close()
```

For a real model or client, initialize it in `__init__` or lazily on first use so each actor owns and reuses that resource. Keep constructors zero-argument; put deployment-specific configuration in a serializable wrapper, class defaults, or the worker environment.

### Scale Backends and Resources

Prove the callable and schema on a small local sample before changing the execution shape:

1. Use `subprocess_task` for a stateless function.
2. Use `subprocess_actor` for local callable-class reuse.
3. Move stateless work to `ray_task` only when distribution helps.
4. Move callable-class work to `ray_actor` for distributed reuse, actor pools, or Ray resource scheduling.

`actor_number` is the Relation actor count; it is not an AI provider's request `concurrency`. On Ray, `gpus` is a scheduler reservation per task or actor. For example, `actor_number=2` with `gpus=1.0` needs two Ray GPU resource slots for full parallelism. On a local subprocess backend, `gpus` does not reserve or isolate a device: workers only inherit process-visible GPUs, and a nonzero declaration may produce a warning. Every worker must be able to import the callable and access its dependencies, model artifacts, data paths, and credentials. Exact backend and resource restrictions are in the [UDF API Reference](/docs/data/reference/udf-api); deployment tasks are covered by [Single-Node Deployment](/docs/data/deploy/single-node), [Ray Cluster Deployment](/docs/data/deploy/ray-cluster), and [GPU Inference](/docs/data/guides/gpu-inference).

### Output, Error, and Side-Effect Contracts

Treat every Relation result as an explicit table contract:

- Return columns and Arrow-compatible values that match the declared schema.
- Include stable IDs, source paths or URIs, and any business keys needed downstream; Relation stages do not automatically retain them.
- Add typed `status` and `error` columns when expected per-row failures should remain reviewable. Let unexpected batch or actor failures fail the query.
- Validate schema, SQL types, row counts or cardinality expectations, nulls, and IDs before increasing `batch_size`, `actor_number`, or data volume.
- Keep UDFs pure when possible. External side effects are not rolled back with the SQL transaction and are not exactly-once; make unavoidable writes idempotent and keyed by stable IDs.
- Write the validated final Relation once instead of writing from inside a UDF.

See [UDF Concepts](/docs/data/concepts/udfs) for the state, retry, and side-effect rationale, and the [UDF API Reference](/docs/data/reference/udf-api) for the exact output and error restrictions.
