本页概述项目开发流程。

## 仓库设置

```bash
git clone --recursive https://github.com/AstroVela/vane.git
cd vane
```

如有需要：

```bash
git submodule update --init --recursive
```

## 构建依赖

项目使用 scikit-build-core 和 CMake。

Python 构建依赖包括：

- `scikit-build-core>=0.11.4`
- `pybind11[global]>=2.6.0`
- `cmake>=3.29.0`
- `ninja>=1.10`

在 Debian/Ubuntu 上：

```bash
sudo apt-get update
sudo apt-get install -y build-essential cmake ninja-build pkg-config curl zip unzip tar flex bison
```

首次构建时，准备代码库内使用的 native 依赖。下面锁定的版本与 `vcpkg.json` 中的 baseline 一致：

```bash
git clone https://github.com/microsoft/vcpkg.git .cache/vcpkg
git -C .cache/vcpkg checkout 44819aa2a6c10e56065e2b0330e7d6c89d1d2574
.cache/vcpkg/bootstrap-vcpkg.sh -disableMetrics
.cache/vcpkg/vcpkg install --x-install-root="$PWD/vcpkg_installed"
```

Vane 的 CMake 配置会自动发现 `vcpkg_installed`，无需设置 `CMAKE_TOOLCHAIN_FILE`。`.cache` 和 `vcpkg_installed` 均已被 Git 忽略。

## 增量开发构建

使用持久化的 Release 构建目录，并安装普通 wheel。请避免 editable mode：Ray worker 会在独立进程中导入 Vane，而 scikit-build-core 的 editable rebuild mode 可能在 worker 启动期间再次调用 CMake。

```bash
export SKBUILD_BUILD_DIR="$PWD/build/release"
export SKBUILD_CMAKE_BUILD_TYPE=Release
python -m pip install . --no-build-isolation -v
```

修改 Python 或 native 源码后，请重新运行安装命令；修改 `vcpkg.json` 后，请重新运行 `vcpkg install`。CMake option 发生变化时，应使用新的构建目录。

## 格式化

源码仓库的 `CONTRIBUTING.md` 描述了如何通过 `scripts/format` 格式化代码。

示例：

```bash
scripts/format root --changed
scripts/format submodule --changed
scripts/format workspace --changed
```

根目录格式化器默认避免扫描 `external/duckdb`。修改 DuckDB 内部实现时，请使用子模块命令。

## 测试

CI 会在 Ubuntu 24.04 和 Python 3.12 上构建并安装 Vane Data，然后运行：

```bash
python -m pytest tests/fast
```

本地聚焦测试：

```bash
python -m pytest tests/fast/test_udf_process.py
```

## 源码区域

- `vane/`：Vane Data 公开 Python API 和 AI 辅助函数。
- `duckdb/execution/`：Python UDF 执行器路由和运行时。
- `duckdb/runners/`：原生与 Ray runner 代码。
- `src/duckdb_py/`：C++ Python 绑定。
- `external/duckdb/`：DuckDB 子模块。
- `examples/`：可运行脚本示例。
- benchmark 工作负载：性能和容错实验。

## 贡献经验法则

保持变更范围聚焦。如果某个变更影响 `map_batches`、`vane.func`、`vane.cls`、`vane.attach_function`、`vane.configure` 或 AI 辅助函数等公开 API，请在同一变更中更新文档和测试。SQL 是推荐的 Expression 入口；应确保 [UDF 概念](/zh-CN/docs/data/concepts/udfs)与 [AI Function 概念](/zh-CN/docs/data/concepts/ai-functions)始终和公开行为一致。
