本页介绍经过测试的本地开发流程。下面的命令会创建本地开发安装，不会生成适合上传到软件包索引的 wheel。

## 从源码构建

当你正在开发 Vane Data、测试尚未发布的改动，或使用的平台没有匹配的 wheel 时，可以从源码构建。Vane 支持 Python 3.10、3.11 和 3.12。推荐使用 Python 3.12，它是主要开发版本，也是 Ubuntu 24.04 x86-64 CI 开发流程使用的版本。其他 native toolchain 和 Linux 发行版可能需要额外配置。

克隆代码库，并拉取子模块：

```bash
git clone --recursive https://github.com/AstroVela/vane.git
cd vane
```

如果检出目录已存在但子模块缺失：

```bash
git submodule update --init --recursive
```

在 Ubuntu 24.04 上，安装 CI 使用的 native 构建工具：

```bash
sudo apt-get update
sudo apt-get install -y \
  autoconf automake bison build-essential ccache curl flex git libtool \
  ninja-build pkg-config tar unzip zip
```

安装 [uv](https://docs.astral.sh/uv/getting-started/installation/)，然后创建并激活 Python 3.12 虚拟环境，再安装代码库声明的构建 dependency group：

```bash
uv venv --python 3.12
source .venv/bin/activate
uv pip install --group build
```

在代码库根目录引导安装锁定的 C++ 依赖：

```bash
bash scripts/bootstrap_vcpkg.sh
```

该脚本会从 `vcpkg.json` 读取精确 baseline，把软件包安装到 `vcpkg_installed`，复用代码库的 vcpkg binary cache，并校验已经提交的 native 依赖许可证清单。Vane 的 CMake 配置会自动发现 `vcpkg_installed`，因此不要设置 `CMAKE_TOOLCHAIN_FILE`。

构建并安装本地、非 editable 的 Release-mode 软件包：

```bash
export SKBUILD_BUILD_DIR="$PWD/build/python-release"
export SKBUILD_CMAKE_BUILD_TYPE=Release

uv pip install . --no-build-isolation
```

这里有意使用 `uv pip install` 而不是 `uv sync`：默认 Ray runner 会在 worker 进程中导入 Vane，而 editable 安装可能在每个 worker 导入时再次调用 CMake。修改源码后，请重新运行安装命令；仅修改 Python 不需要重新编译 native 代码，native 改动则会复用增量构建目录。修改 `vcpkg.json` 后，请重新运行 `scripts/bootstrap_vcpkg.sh`。如果 CMake option 发生变化，请选择新的 `SKBUILD_BUILD_DIR`，或在重新构建前删除旧构建目录。

## 格式化

源码仓库的 `CONTRIBUTING.md` 描述了如何通过 `scripts/format` 格式化代码。请先安装根目录格式化 hooks：

```bash
uv pip install pre-commit
```

示例：

```bash
scripts/format root --changed
scripts/format submodule --changed
scripts/format workspace --changed
```

根目录格式化器默认避免扫描 `external/duckdb`。修改 DuckDB 内部实现时，请使用子模块命令。

## 测试

安装测试 dependency group，然后运行与 CI 相同的 base-installation gate：

```bash
uv pip install --group test
scripts/run_release_tests.sh
```

更广泛的兼容性测试和本地聚焦测试可以分别运行：

```bash
python -m pytest tests/fast
python -m pytest tests/fast/test_udf_process.py
```

部分兼容性测试需要可选服务、凭据、模型权重或 GPU；缺少相应环境时，这些测试应给出明确原因并跳过。
