---
title: Ray 集群
---

当你需要 distributed scan、distributed UDF 或多个 GPU actor 时，使用 Ray runner。

## 选择 Ray runner

```python
import vane

vane.configure(runner="ray")
```

or:

```bash
export VANE_RUNNER=ray
```

## Ray initialization

当 Ray 尚未初始化时，Ray runner 会初始化 Ray。

在用户 workflow 中，如果需要指定 cluster address，请使用标准 Ray environment variable 配置 Ray，或在执行 Vane Data 前先初始化 Ray。

## Worker environment

Vane Data 会将相关 environment variable 传播到 worker，包括：

- `VANE_*`
- `PYTHONPATH`
- `PYTHONWARNINGS`
- `AWS_*`
- `S3FS_*`
- `DUCKDB_*`
- `RAY_ADDRESS`

请在创建 connection 或运行 pipeline 前设置这些变量。

## Distributed UDF

Stateless：

```python
out = rel.map_batches(
    batch_fn,
    schema={"id": "BIGINT", "out": "VARCHAR"},
    batch_size=1024,
    execution_backend="ray_task",
)
```

Stateful/GPU：

```python
out = rel.map_batches(
    ModelActor,
    schema={"id": "BIGINT", "label": "VARCHAR"},
    batch_size=64,
    execution_backend="ray_actor",
    gpus=1,
    concurrency=4,
)
```

当 downstream stage 需要将输出 join 回 source row 时，请从 distributed UDF 返回 identifier 或 metadata。

## S3-compatible storage

设置 credential 和 DuckDB S3 setting，使 driver 和 worker 都能访问相同数据。

```bash
export AWS_ACCESS_KEY_ID=...
export AWS_SECRET_ACCESS_KEY=...
export AWS_REGION=us-east-1
```

对于 MinIO 或其他 S3-compatible service，请通过 DuckDB setting 或 job 使用的 environment 配置 endpoint 和 path style。

## 实用检查

扩展规模前：

- 使用 `subprocess_task` 或 `subprocess_actor` 在本地运行同一个 UDF。
- 确认包含 UDF 的 module 可以在 worker 上 import。
- 确认 model file 在每个 worker 上可用，或可以下载。
- 确认 output path 是共享路径或远端路径。
- 检查 Ray 报告的 CPU/GPU resource 足以支撑你的 `concurrency` 和 `gpus` 选择。

## 关闭

Ray runner 拥有 actor pool，并暴露内部 shutdown method。在普通 script 中，关闭 connection，并让 process 干净退出即可。对于 long-running process，请显式关闭你创建的 resource。
