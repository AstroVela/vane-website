---
title: Ray 集群
---

当你需要分布式扫描、分布式 UDF 或多个 GPU Ray actor 时，使用 Ray runner。

## 选择 Ray runner

```python
import vane

vane.configure(runner="ray")
```

或：

```bash
export VANE_RUNNER=ray
```

## Ray 初始化

当 Ray 尚未初始化时，Ray runner 会初始化 Ray。

在用户 workflow 中，如果需要指定集群地址，请使用标准 Ray 环境变量配置 Ray，或在执行 Vane Data 前先初始化 Ray。

## worker 环境

Vane Data 会将相关环境变量传播到 worker，包括：

- `VANE_*`
- `PYTHONPATH`
- `PYTHONWARNINGS`
- `AWS_*`
- `S3FS_*`
- `DUCKDB_*`
- `RAY_ADDRESS`

请在创建连接或运行 pipeline 前设置这些变量。

## 分布式 UDF

无状态：

```python
out = rel.map_batches(
    batch_fn,
    schema={"id": "BIGINT", "out": "VARCHAR"},
    batch_size=1024,
    execution_backend="ray_task",
)
```

Actor-backed/GPU：

```python
out = rel.map_batches(
    ModelActor,
    schema={"id": "BIGINT", "label": "VARCHAR"},
    batch_size=64,
    execution_backend="ray_actor",
    gpus=1,
    actor_number=4,
)
```

当下游阶段需要将输出关联回源行时，请从分布式 relation UDF 返回标识符或元数据。如果每行结果应留在源列旁，请使用保持行数的 expression。

## 兼容 S3 的存储

设置凭证和 DuckDB S3 设置，使 driver 和 worker 都能访问相同数据。

```bash
export AWS_ACCESS_KEY_ID=...
export AWS_SECRET_ACCESS_KEY=...
export AWS_REGION=us-east-1
```

对于 MinIO 或其他兼容 S3 的服务，请通过 DuckDB 设置或作业使用的环境配置端点和路径风格。

## 实用检查

扩展规模前：

- 使用 `subprocess_task` 或 `subprocess_actor` 在本地运行同一个 UDF。
- 确认包含 UDF 的模块可以在 worker 上导入。
- 确认模型文件在每个 worker 上可用，或可以下载。
- 确认输出路径是共享路径或远端路径。
- 检查 Ray 报告的 CPU/GPU 资源足以支撑你的 `actor_number` 和 `gpus` 选择。
- 把 `vane.cls` 状态视为单 actor、query scope、actor-local 且临时的状态；actor 丢失后不会恢复。

Relation、Expression、状态和 GPU 的能力边界见 [UDF API 参考](/zh-CN/docs/data/reference/udf-api)。

## 关闭

Ray runner 拥有 actor 池，并暴露内部关闭方法。在普通脚本中，关闭连接，并让进程干净退出即可。对于长时间运行的进程，请显式关闭你创建的资源。
