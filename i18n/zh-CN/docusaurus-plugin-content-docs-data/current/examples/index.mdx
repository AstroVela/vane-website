这些示例展示如何将 Vane Data SQL、Python UDF、AI helper 和 Ray 支持的执行组合成实用的数据准备 workflow。

如果你需要理解 workflow 形态，请先阅读下面的文档页。如果需要可运行参数、样本数据行为或实现细节，请阅读源脚本。

## 如何改造示例

改造示例时：

1. 如果脚本提供样本或合成源模式，请从它开始。
2. 将源加载与转换分离。
3. 保持每个 UDF 的输出 schema 显式。
4. 保留稳定标识符，以便模型输出可以关联回源行。
5. 只有在本地样本正常后，再迁移到真实数据。
6. 将 benchmark 脚本视为性能参考，而不是产品 API。
7. 不要从示例名称推断当前未支持的连接器。

## 扩展路径

大多数示例都应先在本地开发。只有在本地数据契约正确后，才将选定阶段迁移到 Ray：

```python
import vane

vane.configure(runner="ray")
```

然后在需要分布式执行的 UDF 或 AI helper 阶段上使用 `ray_task` 或 `ray_actor`。
