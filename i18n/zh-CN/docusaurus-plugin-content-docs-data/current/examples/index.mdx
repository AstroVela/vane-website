如果你需要理解 workflow shape，请先阅读下面的文档页。如果需要可运行参数、sample data 行为或实现细节，请阅读 source script。

## 如何改造示例

改造示例时：

1. 如果 script 提供 sample 或 synthetic source mode，请从它开始。
2. 将 source loading 与 transformation 分离。
3. 保持每个 UDF output schema 显式。
4. 保留稳定 identifier，以便 model output 可以 join 回 source row。
5. 只有在本地 sample 正常后，再迁移到真实数据。
6. 将 benchmark script 视为 performance reference，而不是 product API。
7. 不要从示例名称推断当前未支持的 connector。

## 扩展路径

大多数示例都应先在本地开发。只有在本地 data contract 正确后，才将选定 stage 迁移到 Ray：

```python
import vane

vane.configure(runner="ray")
```

然后在需要 distributed execution 的 UDF 或 AI helper stage 上使用 `ray_task` 或 `ray_actor`。
