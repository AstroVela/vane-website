title: 训练数据流水线
---

本页以 Vane 代码仓库中的三个文本处理脚本为权威实现：

- `examples/common_crawl.py`
- `examples/minhash_dedupe.py`
- `examples/llms_red_pajamas.py`

它们共同覆盖网页文本抽取与分块、近重复删除、embedding 和语义匹配。

## 创建干净环境

先按照[安装](/zh-CN/docs/data/quickstart/installation)页面安装 `uv`，再使用 sparse checkout，把示例脚本检出到新目录：

```bash
git clone --depth 1 --filter=blob:none --sparse \
  https://github.com/AstroVela/vane.git vane-examples
cd vane-examples
git sparse-checkout set examples

uv venv --python 3.12
source .venv/bin/activate
uv pip install vane-ai numpy pyarrow
uv pip install torch
uv pip install sentence-transformers transformers
```

Sparse checkout 不会检出 `vane/` 和 `duckdb/` 源码目录，因此 Python 进程和本地 Ray worker 都会使用已安装的 wheel。Common Crawl 和 Red Pajamas 的 embedding 阶段需要这些模型包；`minhash_dedupe.py` 不需要它们。要使用 GPU 加速，请根据操作系统、GPU 和驱动安装兼容的 CUDA PyTorch build；默认包不适用时，请通过 [PyTorch Start Locally](https://pytorch.org/get-started/locally/)选择安装命令。确认 `python -c "import torch; print(torch.cuda.is_available())"` 输出 `True` 后，Vane 的 Transformers provider 会自动申请一个 Ray GPU。首次生成 embedding 时会下载 `sentence-transformers/all-MiniLM-L6-v2`。只有在本地缓存或传入的模型路径已经包含模型后，才使用 `--local-files-only`。

如果 PyPI 下载较慢，可以在安装命令末尾追加清华镜像：

```bash
uv pip install torch -i https://pypi.tuna.tsinghua.edu.cn/simple
```

镜像只改变包的下载来源，不会替用户选择与当前机器兼容的 CUDA build。

## 运行三个样本

在代码仓库根目录运行：

```bash
python examples/common_crawl.py --source sample --limit 5
python examples/minhash_dedupe.py --source sample --limit 10
python examples/llms_red_pajamas.py --source sample --limit 6
```

使用当前内置样本时，脚本会报告：

| 脚本 | 校验结果 |
| --- | --- |
| `common_crawl.py` | 5 条源记录、3 个解码后的英文页面、7 个文本 chunk，以及 384 维 embedding |
| `minhash_dedupe.py` | 10 条输入，保留 5 条代表记录 |
| `llms_red_pajamas.py` | 6 个已生成 embedding 的问题，以及 3 条语义匹配 |

终端表格的具体格式可能调整，但脚本会在把源行与模型输出组合前校验行数。

## Common Crawl workflow

`common_crawl.py` 包含以下阶段：

1. 加载内置 WARC 形态记录、本地 WET/WARC 文件或 WET URL。
2. 保留 `WARC-Type = conversion` 记录。
3. 使用 `DecodeWarcBatch` 解码内容并解析语言元数据。
4. 保留目标语言，再使用 `ChunkTextBatch` 切分文本。
5. 通过 Transformers provider 调用 `vane.ai.embed_text`。
6. 写出过滤后的页面、chunk 和 embedding。

默认输出位于 `examples/output/common_crawl/`：

- `filtered_pages.csv`
- `chunks.csv`
- `chunk_embeddings.csv`
- `chunk_embeddings.parquet`

不安装模型 runtime 也可以验证抽取和分块：

```bash
uv pip install vane-ai numpy pyarrow
python examples/common_crawl.py \
  --source sample \
  --limit 5 \
  --skip-embeddings
```

使用真实的本地或远程 WET 源：

```bash
python examples/common_crawl.py \
  --source wet-file \
  --wet-path /path/to/sample.warc.wet.gz \
  --limit 100

python examples/common_crawl.py \
  --source wet-url \
  --wet-url https://example.org/sample.warc.wet.gz \
  --limit 100
```

## MinHash 去重 workflow

`minhash_dedupe.py` 会标准化每个文本块，计算确定性的 MinHash 签名，创建 LSH 候选对，默认再用精确 Jaccard 相似度校验候选，构建连通分量，并为每个分量保留一条代表记录。

默认输出目录 `examples/output/minhash_dedupe/` 包含带标注的行、保留行、重复行、cluster、候选对，以及发生碰撞的 LSH bucket。

使用 CSV 输入：

```bash
python examples/minhash_dedupe.py \
  --source csv \
  --csv-path /path/to/documents.csv \
  --text-column text \
  --id-column document_id \
  --limit 1000
```

使用本地 HTML 文件：

```bash
python examples/minhash_dedupe.py \
  --source html-glob \
  --html-glob '/path/to/pages/*.html' \
  --limit 1000
```

## Red Pajamas 语义匹配 workflow

`llms_red_pajamas.py` 为 StackExchange 形态的问题文本生成 embedding，将低分问题与较高分候选分开，计算余弦相似度，并打印 top 匹配。需要保存结果时运行：

```bash
python examples/llms_red_pajamas.py \
  --source sample \
  --limit 6 \
  --output-csv examples/output/red_pajamas_matches.csv
```

真实数据源模式会读取脚本中配置的 Red Pajamas StackExchange JSONL 路径：

```bash
python examples/llms_red_pajamas.py \
  --source redpajama \
  --limit 1000
```

## 默认 Ray 执行

这些脚本不会调用 `vane.configure(...)`，也不会传入 `execution_backend`。Ray 已经是默认 runner。内置样本行会表示为 DuckDB `VALUES` relation，让执行计划可以传到 Ray worker；它们不会被捕获为仅当前进程可见的 Arrow scan。

如需使用已有集群，请在启动脚本前设置 `RAY_ADDRESS`。每个 worker 都必须安装相同的 Vane 和 Transformers 包，并访问相同的模型缓存或模型路径。无需修改代码。

## 生产改造建议

- 将源加载与转换阶段分离。
- 为每个 batch UDF 保持显式 schema。
- 在分块和去重过程中保留稳定的源 ID。
- 追加 embedding 或 prompt 输出前校验行数。
- 先使用有界输入，再提高 `--limit` 并调节脚本已有的 `--batch-size` 参数。
- 以 `examples/` 下的文件作为实现约定，不要从旧的外部片段复制 backend 或 runner 参数。
