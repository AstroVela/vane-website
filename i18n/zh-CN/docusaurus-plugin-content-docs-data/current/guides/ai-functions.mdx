title: AI 函数

Vane Data 面向多模态 AI 工作负载设计。AI Function 让你能直接在数据上运行模型：生成 embedding、做文本分类，或调用大语言模型、多模态模型。

对于 embedding、文本分类和提示词调用等常见任务，AI Function 是合适的起点。当主要工作是对大量文档分块并生成 embedding 时，使用 [大规模 Embedding](/zh-CN/docs/data/guides/embeddings-at-scale)。对于 vLLM 或 GPU 加速的 batch 提示词调用，使用 [GPU 推理](/zh-CN/docs/data/guides/gpu-inference)。当你需要完全控制模型加载、重试、输出 schema、预处理或逐行错误列时，使用 [自定义 Python UDF](/zh-CN/docs/data/guides/custom-python-udfs)。

## 输入 relation

先准备一个包含以下列的 relation：

- 稳定的业务 ID，例如 `document_id`、`ticket_id`、`case_id` 或 `line_item_id`。
- 模型所需的文本列或字段。
- 便于后续审查异常输出的源元数据，例如 `source_uri`、`media_type`、`document_type` 或 `created_at`。

```python
import vane

con = vane.connect()

source = con.sql("""
    select
        document_id,
        document_type,
        text,
        source_uri
    from read_parquet('data/documents/*.parquet')
    where text is not null
    order by document_id
    limit 100
""")

source.to_table("source_sample")
```

保持第一个样本小且有序。稳定排序可以让你以确定性的方式校验辅助函数输出，并将其与源行组合。

## 保留 ID

`embed_text(...)`、`classify_text(...)` 和 `prompt(...)` 会返回已配置的输出列。如果最终结果还需要 ID 和源列，请先验证行数和顺序，再显式地将输出列与源行组合。

```python
source_table = source.to_arrow_table()
```

对于下面的每个任务，比较行数，并且只有在检查通过后，才将新列追加到 `source_table`：

```python
output_table = output_only.to_arrow_table()

if output_table.num_rows != source_table.num_rows:
    raise ValueError("AI output row count does not match source row count")

with_output = con.from_arrow(
    source_table.append_column("ai_output", output_table["ai_output"])
)
```

将 `ai_output` 替换为该任务实际使用的输出列名。如果不希望单独执行重组步骤，可以编写自定义 `map_batches(...)` UDF，在一个 schema 中同时返回稳定 ID、源字段、状态、错误列和模型输出。

## Embed Text

当每个输入行都应该获得一个向量时，使用 `embed_text`。对于长文档，先做分块，并使用 [大规模 Embedding](/zh-CN/docs/data/guides/embeddings-at-scale)。

```python
embedding_only = source.embed_text(
    "text",
    provider="openai",
    model="text-embedding-3-small",
    output_column="embedding",
    batch_size=64,
    concurrency=4,
    execution_backend="subprocess_actor",
)
```

校验项：

- 输入列：过滤后 `text` 应为非 null。
- 输出列：`embedding` 应存在，并且向量长度应符合该模型的预期。
- ID 保留：只有在行数校验通过后，才将 embedding 列与 `document_id` 组合。
- 结果审查：检查少量行，确认空文本或过短文本没有被意外生成 embedding。

```python
embedding_table = embedding_only.to_arrow_table()

if embedding_table.num_rows != source_table.num_rows:
    raise ValueError("embedding row count mismatch")

if embedding_table.num_rows == 0:
    raise ValueError("empty embedding sample")

if not embedding_table["embedding"][0].as_py():
    raise ValueError("first embedding is empty")

embedded = con.from_arrow(
    source_table.append_column("embedding", embedding_table["embedding"])
)
```

## Classify Text

当标签集合较小，并且每个输入行都应获得一个标签时，使用 `classify_text`。

```python
label_only = source.classify_text(
    "text",
    labels=["invoice", "claim", "memo", "other"],
    provider="transformers",
    model="facebook/bart-large-mnli",
    output_column="ai_label",
    batch_size=16,
    execution_backend="subprocess_actor",
)
```

校验项：

- 输入列：`text` 应包含你希望分类器看到的文本。
- 输出列：`ai_label` 应只包含你提供的标签之一。
- ID 保留：行数校验后，将 `ai_label` 追加到源表。
- 结果审查：按标签抽样检查，确认标签集合是否过宽、过窄，或缺少业务分类。

```python
label_table = label_only.to_arrow_table()

if label_table.num_rows != source_table.num_rows:
    raise ValueError("classification row count mismatch")

labels = set(label_table["ai_label"].to_pylist())
allowed = {"invoice", "claim", "memo", "other"}
unexpected = labels - allowed
if unexpected:
    raise ValueError(f"unexpected labels: {[str(x) for x in unexpected]}")

classified = con.from_arrow(
    source_table.append_column("ai_label", label_table["ai_label"])
)
```

## Prompt

当输出需要由逐行提示词生成时，使用 `prompt`。第一个提示词应保持范围明确，并便于人工检查。

```python
prompts = con.sql("""
    select
        document_id,
        document_type,
        source_uri,
        'Summarize this document in one sentence: ' || text as review_prompt
    from source_sample
""")

response_only = prompts.prompt(
    "review_prompt",
    provider="openai",
    model="gpt-4o-mini",
    system_message="Return concise business review notes.",
    output_column="review_note",
    batch_size=8,
    concurrency=4,
    execution_backend="subprocess_actor",
)
```

校验项：

- 输入列：`review_prompt` 只应包含模型应该看到的字段。
- 输出列：`review_note` 应为非空，并且足够短，便于审查。
- ID 保留：将输出与 `document_id`、`document_type` 和 `source_uri` 组合。
- 结果审查：在增大 `limit`、`batch_size` 或 `concurrency` 前，先人工检查小样本。

```python
prompt_table = prompts.to_arrow_table()
response_table = response_only.to_arrow_table()

if response_table.num_rows != prompt_table.num_rows:
    raise ValueError("prompt row count mismatch")

empty = [x for x in response_table["review_note"].to_pylist() if not x]
if empty:
    raise ValueError("prompt returned empty responses")

reviewed = con.from_arrow(
    prompt_table.append_column("review_note", response_table["review_note"])
)
```

对于结构化输出，使用 `return_format` 配合 Pydantic 模型，并在写入最终 relation 前校验 JSON 字符串。

## Provider 与执行选择

选择能够完成当前任务的最小 provider 配置。

| 选择 | 适用场景 | 需要关注 |
| --- | --- | --- |
| 托管 provider | 需要托管模型，并且可以使用外部 API 调用。 | 凭证、成本、延迟、速率限制，以及 provider 专用的输出行为。 |
| 本地模型 | 需要基于已安装模型库的本地执行。 | 模型下载、依赖版本、内存使用，以及 actor 初始化时间。 |
| vLLM 或 GPU 模型 | 需要 GPU 加速的提示词吞吐量或私有 LLM 推理。 | vLLM 路径和 GPU 专用调优请参考 [GPU 推理](/zh-CN/docs/data/guides/gpu-inference)。 |

建议显式设置的最小配置：

- `provider`：例如 `openai`、`transformers` 或 `vllm`。
- `model`：准确的托管或本地模型标识符。
- `batch_size`：每次模型调用或 batch 处理的行数。
- `concurrency`：在支持时，并行运行的 worker 或 actor 调用数量。
- `execution_backend`：从 `subprocess_actor` 开始；只有在本地样本正确后，再使用 `ray_actor`。

一次只调整一个设置。更高的 `batch_size` 或 `concurrency` 可以提升吞吐量，但也可能增加内存使用、触发 provider 速率限制，或让失败更难隔离。

## 下一步

- 使用 [大规模 Embedding](/zh-CN/docs/data/guides/embeddings-at-scale) 处理文档分块、稳定的分块 ID、embedding 输出 schema，以及更大规模的 embedding 作业。
- 使用 [GPU 推理](/zh-CN/docs/data/guides/gpu-inference) 处理基于 vLLM 的提示词 batch、GPU actor 复用和 Ray GPU 执行。
- 当 AI Function 范围过窄，或输出必须由一个 batch 函数同时包含 ID、源字段、状态、错误和模型输出时，使用 [自定义 Python UDF](/zh-CN/docs/data/guides/custom-python-udfs)。
