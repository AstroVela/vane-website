---
title: 自定义 Python UDF
---

当 Vane Data pipeline 需要 SQL 或内置 AI Function 不适合承载的 Python 逻辑时，使用 Custom Python UDF：外部 library、自定义 rule、media processing、model preprocessing、自定义 inference、private client，或完全可控的 output schema。

这篇文档聚焦 UDF 有哪些类型，以及应该如何选择。

## 选择 UDF 类型

先选择 callable 形态。Execution backend 是另一层选择，它决定 callable 在哪里运行。

| UDF 类型 | API | Callable 形态 | 本地 backend | Ray backend | 适用场景 |
| --- | --- | --- | --- | --- | --- |
| 无状态 batch UDF | `map_batches` | Python function | `subprocess_task` | `ray_task` | 逻辑不需要可复用 model、client、cache 或其他状态。 |
| 有状态 batch UDF | `map_batches` | Callable class | `subprocess_actor` | `ray_actor` | setup 成本高，需要跨 batch 复用。 |
| 行扩展 UDF | `flat_map` | Row function | `subprocess_task` | `ray_task` | 一个输入行可能产生零行、一行或多行输出。 |
| 标量 UDF | `map` | Scalar function | `subprocess_task` | `ray_task` | 只需要简单的逐行标量转换。 |

大多数生产 UDF 应该先判断是无状态 batch UDF 还是有状态 batch UDF。需要 chunking 这类行扩展时使用 `flat_map`，只有简单标量 enrichment 才使用 `map`。

## 无状态 Batch UDF

当每个 batch 都可以独立处理时，使用无状态 batch UDF：

- text cleanup
- rule-based labeling
- metadata check
- lightweight parsing
- deterministic feature extraction
- 不需要可复用状态的 Python library call

无状态 batch UDF 是一个 Python function，接收 `pyarrow.Table`，返回 `pyarrow.Table`。

```python
import re

import pyarrow as pa
import vane

con = vane.connect()

documents = con.sql("""
    select
        cast(document_id as varchar) as document_id,
        source_uri,
        media_type,
        document_type,
        text
    from read_parquet('data/documents/*.parquet')
    where text is not null
    order by document_id
    limit 100
""")

SUPPORTED_MEDIA_TYPES = {"application/pdf", "text/plain", "text/html"}
RISK_TERMS = ("missing signature", "expired", "redacted incorrectly")


def prepare_document_text(batch: pa.Table) -> pa.Table:
    document_ids = batch.column("document_id").to_pylist()
    source_uris = batch.column("source_uri").to_pylist()
    media_types = batch.column("media_type").to_pylist()
    document_types = batch.column("document_type").to_pylist()
    texts = batch.column("text").to_pylist()

    normalized_text = []
    text_length = []
    needs_review = []
    review_reason = []
    status = []
    error = []

    for media_type, text in zip(media_types, texts, strict=False):
        try:
            clean = re.sub(r"\s+", " ", str(text or "")).strip()
            lower = clean.lower()

            normalized_text.append(clean)
            text_length.append(len(clean))

            if media_type not in SUPPORTED_MEDIA_TYPES:
                needs_review.append(True)
                review_reason.append("unsupported_media_type")
            elif not clean:
                needs_review.append(True)
                review_reason.append("empty_text")
            elif any(term in lower for term in RISK_TERMS):
                needs_review.append(True)
                review_reason.append("risk_term")
            else:
                needs_review.append(False)
                review_reason.append("none")
            status.append("ok")
            error.append("")
        except Exception as exc:
            normalized_text.append("")
            text_length.append(0)
            needs_review.append(True)
            review_reason.append("udf_error")
            status.append("failed")
            error.append(f"prepare_document_text: {type(exc).__name__}: {exc}")

    return pa.table({
        "document_id": document_ids,
        "source_uri": source_uris,
        "media_type": media_types,
        "document_type": document_types,
        "normalized_text": normalized_text,
        "text_length": text_length,
        "needs_review": needs_review,
        "review_reason": review_reason,
        "status": status,
        "error": error,
    })


prepared_schema = {
    "document_id": "VARCHAR",
    "source_uri": "VARCHAR",
    "media_type": "VARCHAR",
    "document_type": "VARCHAR",
    "normalized_text": "VARCHAR",
    "text_length": "BIGINT",
    "needs_review": "BOOLEAN",
    "review_reason": "VARCHAR",
    "status": "VARCHAR",
    "error": "VARCHAR",
}

prepared = documents.map_batches(
    prepare_document_text,
    schema=prepared_schema,
    batch_size=64,
    execution_backend="subprocess_task",
)

prepared.show()
```

返回的 table 同时包含新字段和后续阶段需要的 source field。`map_batches` 返回 UDF 产出的列，不会自动保留 upstream column。

增大 batch size 或迁移到 Ray 之前，先校验这个无状态 UDF：

```python
prepared.to_table("prepared_documents")

summary = con.sql("""
    select
        document_type,
        media_type,
        needs_review,
        review_reason,
        count(*) as rows
    from prepared_documents
    group by document_type, media_type, needs_review, review_reason
    order by document_type, media_type, needs_review, review_reason
""")

summary.show()
```

本地输出正确后，同一个 function 可以用 `ray_task` 运行：

```python
import vane

vane.configure(runner="ray")

ray_prepared = documents.map_batches(
    prepare_document_text,
    schema=prepared_schema,
    batch_size=256,
    execution_backend="ray_task",
)
```

只有在本地 contract 正确后才使用 `ray_task`。Ray 会额外引入 worker import、dependency、storage 和 resource 等变量，可能掩盖普通 UDF bug。

## 有状态 Batch UDF

当 setup 需要跨 batch 复用时，使用有状态 batch UDF：

- model 或 tokenizer loading
- API client setup
- database 或 object-store client setup
- lookup table 或 cache
- GPU model state

有状态 UDF 是 callable class。传给 `map_batches` 的应该是 class 本身，而不是已经创建好的 instance。Actor backend 会在 worker 内构造 class，并在多个 batch 间复用同一个 object。

Actor class 不应有必填 constructor argument。可以把配置放在环境变量、class-level default、带默认值的 optional argument，或 no-argument wrapper class 中。

```python
import pyarrow as pa


class PromptBuilder:
    def __init__(self) -> None:
        self.template_by_type = {
            "claim": "Extract claim decision and one short reason:",
            "invoice": "Extract invoice number, supplier, and total:",
            "policy": "Summarize policy issue and required follow-up:",
        }

    def __call__(self, batch: pa.Table) -> pa.Table:
        document_ids = batch.column("document_id").to_pylist()
        source_uris = batch.column("source_uri").to_pylist()
        document_types = batch.column("document_type").to_pylist()
        texts = batch.column("normalized_text").to_pylist()
        upstream_status = batch.column("status").to_pylist()
        upstream_error = batch.column("error").to_pylist()

        prompts = []
        prompt_template = []
        status = []
        error = []

        for document_type, text, previous_status, previous_error in zip(
            document_types,
            texts,
            upstream_status,
            upstream_error,
            strict=False,
        ):
            if previous_status != "ok":
                prompt_template.append("")
                prompts.append("")
                status.append("skipped")
                error.append(previous_error)
                continue

            template = self.template_by_type.get(document_type, "Summarize this document:")
            prompt_template.append(template)
            prompts.append(f"{template}\n\n{text}")
            status.append("ok")
            error.append("")

        return pa.table({
            "document_id": document_ids,
            "source_uri": source_uris,
            "prompt_template": prompt_template,
            "prompt": prompts,
            "status": status,
            "error": error,
        })


prompt_schema = {
    "document_id": "VARCHAR",
    "source_uri": "VARCHAR",
    "prompt_template": "VARCHAR",
    "prompt": "VARCHAR",
    "status": "VARCHAR",
    "error": "VARCHAR",
}

prompts = prepared.map_batches(
    PromptBuilder,
    schema=prompt_schema,
    batch_size=64,
    execution_backend="subprocess_actor",
    concurrency=2,
)
```

在真实 model UDF 中，请在 `__init__` 中或第一次调用时 lazy load model、tokenizer 或 client，而不是在 module import 时加载。这样可以保持 import 轻量，并让每个 subprocess 或 Ray actor 拥有自己的状态。

如果需要分布式 actor 复用，把 backend 切换到 `ray_actor`：

```python
ray_prompts = prepared.map_batches(
    PromptBuilder,
    schema=prompt_schema,
    batch_size=128,
    execution_backend="ray_actor",
    concurrency=4,
)
```

对于 GPU-backed state，请按 actor 请求 GPU resource：

```python
class GpuEmbedder:
    def __init__(self) -> None:
        # 在这里加载 tokenizer/model，使每个 actor 拥有自己的 GPU state。
        self.model_name = "local-or-mounted-model"

    def __call__(self, batch: pa.Table) -> pa.Table:
        document_ids = batch.column("document_id").to_pylist()
        source_uris = batch.column("source_uri").to_pylist()
        texts = batch.column("normalized_text").to_pylist()

        embeddings = [[0.0, 0.0, 0.0] for _ in texts]
        return pa.table({
            "document_id": document_ids,
            "source_uri": source_uris,
            "embedding": embeddings,
            "status": ["ok"] * len(document_ids),
            "error": [""] * len(document_ids),
        })


embedded = prepared.map_batches(
    GpuEmbedder,
    schema={
        "document_id": "VARCHAR",
        "source_uri": "VARCHAR",
        "embedding": "FLOAT[]",
        "status": "VARCHAR",
        "error": "VARCHAR",
    },
    batch_size=32,
    execution_backend="ray_actor",
    concurrency=2,
    cpus=0.0,
    gpus=1.0,
)
```

`gpus=1.0` 是每个 actor 的资源请求。使用 `concurrency=2` 时，Ray 需要两个可用 GPU slot 才能满并行运行。更多 GPU execution pattern，请参考 [GPU Inference](/zh-CN/docs/data/guides/gpu-inference)。

每个 Ray worker 都必须能够 import 该 callable，并访问相同的数据路径、model file、credential 和 Python dependency。

## 行扩展 UDF

当一个输入行可能产生零行、一行或多行输出时，使用 `flat_map`。典型场景包括 document chunking、frame extraction summary、nested record normalization、URL expansion，以及通过返回空结果来过滤行。

Callable 接收一行 Python `dict`。它可以返回 `None`、一个 output `dict`，或 output dict 的 iterable。

```python
def chunk_document(row: dict) -> list[dict]:
    text = str(row.get("normalized_text") or "")
    chunks = [text[index:index + 1000] for index in range(0, len(text), 1000)]

    return [
        {
            "document_id": row["document_id"],
            "chunk_id": f"{row['document_id']}:{position}",
            "chunk_text": chunk,
        }
        for position, chunk in enumerate(chunks)
        if chunk
    ]


chunks = prepared.flat_map(
    chunk_document,
    schema={
        "document_id": "VARCHAR",
        "chunk_id": "VARCHAR",
        "chunk_text": "VARCHAR",
    },
    execution_backend="subprocess_task",
)
```

声明的 schema 会从每个 output dict 中选择 key。缺失的 key 会变成 null，额外 key 会被忽略。

## 标量 UDF

当只需要简单的逐行标量转换，并且 batch UDF 显得过重时，使用 `map`。

```python
def normalize_document_type(document_type: str) -> str:
    return str(document_type or "unknown").strip().lower()


normalized_types = documents.project("document_type").map(
    normalize_document_type,
    return_type="VARCHAR",
    execution_backend="subprocess_task",
)
```

对于 model call、media decoding、embedding、LLM call，或任何可以从 batching 获益的操作，优先使用 `map_batches`。

## 共同 Output Contract

请把 UDF output 视为 table contract。

| 主题 | 建议 |
| --- | --- |
| Returned columns | 对 `map_batches` 和 `flat_map`，精确返回 `schema` 中声明的列。 |
| Source fields | 返回后续所需的稳定 ID、source URI/path、media type、business key、文本或 metadata。 |
| Row diagnostics | 当失败行需要保持可审查时，返回显式 `status` 和 `error` 列。 |
| Types | 使用 Arrow-compatible value：string、number、boolean、bytes、list、struct 或 Arrow array。 |
| Nulls | 保持 null 行为显式，尤其是 numeric、bytes、list 和 model-output 列。 |
| Row count | `map_batches` UDF 通常每个输入行返回一个输出行。当输出可能为零行或多行时，使用 `flat_map`。 |

如果 Arrow 无法推断你需要的类型，请构造显式 array：

```python
return pa.table({
    "document_id": pa.array(document_ids, type=pa.string()),
    "review_reason": pa.array(review_reason, type=pa.string()),
})
```

避免返回 model class、file handle、自定义 dataclass 或 raw exception 等 Python object。返回前请将它们转换为 string、number、bytes、list、struct、JSON string 或显式 Arrow array。

## Backend 与扩展

Backend 选择应该跟随 UDF 类型：

| Backend | Callable 形态 | 适用场景 |
| --- | --- | --- |
| `subprocess_task` | Function | 第一个 stateless UDF、本地隔离、CPU transform。 |
| `subprocess_actor` | Callable class | 不使用 Ray 的本地 model 或 client 复用。 |
| `ray_task` | Function | 本地正确性验证后，再执行 distributed stateless batch work。 |
| `ray_actor` | Callable class | Distributed model/client 复用、GPU placement，或更大的 actor pool。 |

唯一有效的 backend name 是 `subprocess_task`、`subprocess_actor`、`ray_task` 和 `ray_actor`。`local`、`subprocess`、`ray=True`、`use_process`、`actor_count`、`prefer_actor` 等旧写法或 shorthand 不是当前 API。

建议按这个顺序推进：

1. 使用 `subprocess_task` 或 `subprocess_actor` 在本地构建 UDF。
2. 在小型真实 sample 上校验 schema、row count、ID、source column 和 error behavior。
3. 输出正确后，在本地增大 `batch_size`。
4. 只有当 distribution 有帮助时，才把无状态 function 迁移到 `ray_task`。
5. 只有当 actor 复用、GPU placement 或 cluster scheduling 是必要条件时，才把有状态 class 迁移到 `ray_actor`。

`batch_size` 控制 Vane 每次 `map_batches` 调用传入多少输入行。Batch size 不应改变结果语义。如果调整 `batch_size` 后输出发生变化，UDF 很可能依赖了隐藏的 batch state。

`concurrency` 支持 `subprocess_task`、`subprocess_actor` 和 `ray_actor`；不要将它与 `ray_task` 一起使用。

## 调试

在将 UDF 放入更大的 pipeline 之前，按以下顺序调试：

1. 选择带有稳定 ID 的小型真实 sample。
2. 直接在 Arrow table 上调用 Python function 或 callable class。
3. 使用较小的 `batch_size`，通过 Vane 运行同一个 callable。
4. 使用 SQL 校验 row count、ID、source column、output type 和 status/error behavior。
5. 在本地增大 `batch_size`。
6. 只有在本地 contract 正确后，才迁移到 Ray。

```python
sample = con.sql("""
    select document_id, source_uri, media_type, document_type, text
    from read_parquet('data/documents/*.parquet')
    where text is not null
    order by document_id
    limit 20
""")

sample_arrow = sample.to_arrow_table()
debug_arrow = prepare_document_text(sample_arrow)

assert len(debug_arrow) == len(sample_arrow)
print(debug_arrow.schema)
print(debug_arrow.to_pydict())
```

然后通过 Vane 运行：

```python
debug_relation = sample.map_batches(
    prepare_document_text,
    schema=prepared_schema,
    batch_size=4,
    execution_backend="subprocess_task",
)

debug_relation.show()
```

## 常见失败

| 失败 | 通常意味着 | 修复方式 |
| --- | --- | --- |
| Schema mismatch | 返回的列或类型与 `schema` 不匹配。 | 打印返回的 Arrow schema，然后对齐 column name 和 Arrow-compatible type。 |
| Missing ID or source columns | UDF 只返回了 derived column。 | 返回 downstream SQL、join 和 review 所需的稳定 ID 与 source field。 |
| Missing status or error fields | UDF 依赖 exception 或 log 作为 row-level diagnostics。 | 当失败行需要保持可审查时，返回显式 `status` 和 `error` 列。 |
| Arrow type incompatibility | UDF 返回了 Arrow 无法序列化的 Python object。 | 将输出转换为 primitive、bytes、list、struct、JSON string 或显式 `pa.array` value。 |
| Wrong callable shape | task backend 收到了 class/instance，或 actor backend 收到了 function/instance。 | 对 `subprocess_task`/`ray_task` 使用 function；对 `subprocess_actor`/`ray_actor` 使用无必填构造参数的 callable class。 |
| Worker import failure | subprocess 或 Ray worker 无法 import 定义 UDF 的 module。 | 将被引用的代码放入可 import 的 module，并在每个 worker 上安装相同 Python dependency。 |
| Model loads at import time | 每次 worker import 都触发重型 setup，或在 actor 启动前失败。 | 将 model、tokenizer 和 client setup 移到 callable class 的 `__init__` 或首次调用中。 |
| Batch works locally but fails on Ray | Worker 无法访问数据、credential、model file 或兼容的 package version。 | 增大数据规模前，先用极小 Ray sample 验证 worker path 和 environment。 |
| Memory spikes | Batch size、media bytes 或 model output 过大。 | 投影更少列，降低 `batch_size`，拆分 stage，或使用 actor 复用 model state。 |

## 相关文档

- 当主要任务是 table-shaped cleanup、join、deduplication、validation 或 output shaping 时，使用 [Structured Transformation](/zh-CN/docs/data/guides/structured-transformation)。
- 当 UDF 需要读取 bytes、解码文件、提取 media feature，或记录 per-stage status 和 error 时，使用 [Multimodal Pipeline](/zh-CN/docs/data/guides/multimodal-pipeline)。
- 对于标准 embedding、classification 或 prompt enrichment，使用 [AI Functions](/zh-CN/docs/data/guides/ai-functions)。当你需要 custom model、custom output schema、external client behavior，或完全控制 preprocessing 与 postprocessing 时，使用 Custom Python UDFs。
- 阅读 [UDFs](/zh-CN/docs/data/concepts/udfs)，了解 `map_batches`、callable class 和 execution backend 背后的概念模型。
