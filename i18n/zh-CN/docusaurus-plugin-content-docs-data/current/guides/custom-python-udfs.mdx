---
title: 自定义 Python UDF
---

本文完成一个真实任务：把文档文本规范化为有类型的审查列，同时保留稳定的文档 ID 与源 URI，校验输出契约，写入 Parquet，并清理已注册的 SQL alias。

精确签名和限制请查阅 [UDF API 参考](/zh-CN/docs/data/reference/udf-api)。SQL-first Expression 模型与 Relation escape hatch 背后的原因请参阅 [UDF 概念](/zh-CN/docs/data/concepts/udfs)。

## 任务目标与前置条件

主要工作流在本地使用内联数据运行。它要求 Vane 已正确安装，但任务本身不需要 Ray 集群、GPU、外部文件、凭据或 PyArrow 代码。结果会写入 `output/document-review.parquet`。

源数据契约包含三个显式类型列：

| 列 | 类型 | 用途 |
| --- | --- | --- |
| `document_id` | `VARCHAR` | 用于行数与身份校验的稳定业务 ID。 |
| `source_uri` | `VARCHAR` | 每条结果都会保留的可追溯源位置。 |
| `text` | `VARCHAR` | 需要规范化、允许为 null 的输入。 |

数值 UDF 也应显式定义同样的类型边界：将 `1.25::DOUBLE` 与 `parameters=["DOUBLE"]`、`return_dtype="DOUBLE"` 配套使用。未转换的 decimal literal 通常是 `DECIMAL`，不应把它与 `DOUBLE` 默认为同一契约。

## 主要工作流：SQL Expression（推荐）

先定义一个原始 Python scalar callable——主要流程不使用 `vane.func` decorator。然后创建 connection 和 source，把有类型的 alias attach 到该 connection，并且只在 SQL `SELECT` projection 中调用它。

```python
import vane

import re
from pathlib import Path


def normalize_review_text(text):
    if text is None:
        return None
    return re.sub(r"\s+", " ", text).strip().lower()


con = vane.connect()
source = con.sql(
    """
    SELECT
        document_id::VARCHAR AS document_id,
        source_uri::VARCHAR AS source_uri,
        text::VARCHAR AS text
    FROM (VALUES
        ('doc-001', 's3://sample/doc-001.txt', ' Missing   signature '),
        ('doc-002', 's3://sample/doc-002.txt', NULL),
        ('doc-003', 's3://sample/doc-003.txt', ' APPROVED ')
    ) AS source_rows(document_id, source_uri, text)
    ORDER BY document_id
    """
)
source.to_table("udf_source_documents")

vane.attach_function(
    normalize_review_text,
    alias="normalize_review_text_sql",
    connection=con,
    parameters=["VARCHAR"],
    return_dtype="VARCHAR",
)

try:
    projected = con.sql(
        """
        SELECT
            document_id,
            source_uri,
            text,
            normalize_review_text_sql(text) AS review_text
        FROM udf_source_documents
        ORDER BY document_id
        """
    )
    projected.to_table("udf_reviewed_documents")
    result = con.table("udf_reviewed_documents")

    expected_ids = [
        row[0]
        for row in con.sql(
            "SELECT document_id FROM udf_source_documents ORDER BY document_id"
        ).fetchall()
    ]
    rows = result.order("document_id").fetchall()
    actual_ids = [row[0] for row in rows]

    assert list(result.columns) == [
        "document_id",
        "source_uri",
        "text",
        "review_text",
    ]
    assert [str(dtype) for dtype in result.types] == [
        "VARCHAR",
        "VARCHAR",
        "VARCHAR",
        "VARCHAR",
    ]
    assert len(rows) == len(expected_ids)
    assert actual_ids == expected_ids
    assert len(set(actual_ids)) == len(expected_ids)

    source_nulls = con.sql(
        "SELECT count(*) FROM udf_source_documents WHERE text IS NULL"
    ).fetchone()[0]
    result_nulls = con.sql(
        "SELECT count(*) FROM udf_reviewed_documents WHERE review_text IS NULL"
    ).fetchone()[0]
    assert result_nulls == source_nulls
    assert rows[0][3] == "missing signature"
    assert rows[2][3] == "approved"

    output_dir = Path("output")
    output_dir.mkdir(parents=True, exist_ok=True)
    result.write_parquet(
        str(output_dir / "document-review.parquet"),
        overwrite=True,
    )
finally:
    try:
        vane.detach_function("normalize_review_text_sql", connection=con)
    finally:
        con.close()
```

Alias 仅限所属 connection，在 v1 中保持行数，并且只能出现在 projection 位置。先用普通 SQL 完成 source 的 filter 和 join；不要把 alias 放进 `WHERE`、`GROUP BY`、`HAVING`、join condition、aggregate argument 或依赖 short-circuit 的 expression。权威的放置与注册规则见 [UDF API 参考](/zh-CN/docs/data/reference/udf-api)。

把结果物化为 `udf_reviewed_documents` 后，校验和最终写入会针对同一份已完成的 UDF 结果，而不会为每项检查重复运行 callable。这些 assertion 会在接受输出之前校验 schema、精确 SQL 类型、行数、去重后的有序 ID、null 传播和代表性值。即使 projection、校验或写入失败，`finally` 仍会从同一个 connection 移除 alias。

SQL 是默认入口，因为 relational selection 保持清晰可见，同时一个有类型的 Python 结果可以与原始列组合。该设计原因见 [UDF 概念](/zh-CN/docs/data/concepts/udfs)。

## 备用工作流：Python Expression

当 expression 需要动态组装、作为 Python 对象复用，或在 notebook 与 library 中组合时，使用 Python 构造方式。它是同一个有类型 projection 模型的备用调用形式，并非不同的输出模型。

### 使用 `vane.func` 的 Scalar Python Expression

下面的任务不注册 SQL alias，也能得到相同的四列、保持行数的 projection：

```python
import vane

import re


@vane.func(return_dtype="VARCHAR")
def normalize_review_text(text):
    if text is None:
        return None
    return re.sub(r"\s+", " ", text).strip().lower()


con = vane.connect()
source = con.sql(
    """
    SELECT
        document_id::VARCHAR AS document_id,
        source_uri::VARCHAR AS source_uri,
        text::VARCHAR AS text
    FROM (VALUES
        ('doc-001', 's3://sample/doc-001.txt', ' Missing   signature '),
        ('doc-002', 's3://sample/doc-002.txt', NULL)
    ) AS source_rows(document_id, source_uri, text)
    """
)

result = source.select(
    vane.col("document_id"),
    vane.col("source_uri"),
    vane.col("text"),
    normalize_review_text(vane.col("text")).alias("review_text"),
)

assert list(result.columns) == [
    "document_id",
    "source_uri",
    "text",
    "review_text",
]
assert len(result.fetchall()) == 2
con.close()
```

### 使用 `vane.func.batch` 的 Arrow Batch Python Expression

当 batch 处理有帮助，但契约仍是输入行旁边的一个有类型输出列时，使用 batch Expression：

```python
import vane

import re
import pyarrow as pa


def normalize_review_batch(table):
    values = table.column("text").to_pylist()
    normalized = [
        None if value is None else re.sub(r"\s+", " ", value).strip().lower()
        for value in values
    ]
    return pa.table({"review_text": pa.array(normalized, type=pa.string())})


con = vane.connect()
source = con.sql(
    """
    SELECT
        document_id::VARCHAR AS document_id,
        source_uri::VARCHAR AS source_uri,
        text::VARCHAR AS text
    FROM (VALUES
        ('doc-001', 's3://sample/doc-001.txt', ' Missing   signature '),
        ('doc-002', 's3://sample/doc-002.txt', NULL)
    ) AS source_rows(document_id, source_uri, text)
    """
)

review_text = vane.func.batch(
    normalize_review_batch,
    inputs={"text": vane.col("text")},
    schema={"review_text": "VARCHAR"},
    row_preserving=True,
).alias("review_text")

result = source.select(
    vane.col("document_id"),
    vane.col("source_uri"),
    vane.col("text"),
    review_text,
)

assert len(result.fetchall()) == 2
con.close()
```

`row_preserving=True` 会检查每个输入行都有一个 batch 输出行，并允许结果与普通列并列。`row_preserving=False` 是 Python batch Expression 的显式例外：它可以改变基数，但 batch 结果必须是唯一的顶层 projection，而且 v1 不允许把它注册为 SQL alias。精确的 batch 与放置契约见 [UDF API 参考](/zh-CN/docs/data/reference/udf-api)。

## Stateful `vane.cls`：窄范围契约

只有可变状态必须在一次 query 的一个 actor 内存活时，才使用 `vane.cls`。v1 要求 `actor_number=1`；状态仅限 query scope、actor 内局部且临时，不 checkpoint，也不提供 exactly-once。下面的 memoizer 会维护 query 内缓存，同时仍返回一个确定的有类型列：

```python
import vane

import re


@vane.cls(actor_number=1, return_dtype="VARCHAR")
class QueryMemoizer:
    def __init__(self):
        self.cache = {}

    def __call__(self, text):
        if text not in self.cache:
            self.cache[text] = re.sub(r"\s+", " ", text).strip().lower()
        return self.cache[text]


memoizer = QueryMemoizer()
con = vane.connect()
source = con.sql(
    """
    SELECT
        document_id::VARCHAR AS document_id,
        text::VARCHAR AS text
    FROM (VALUES
        ('doc-001', ' Repeat   me '),
        ('doc-002', ' Repeat   me ')
    ) AS source_rows(document_id, text)
    """
)

result = source.select(
    vane.col("document_id"),
    memoizer(vane.col("text")).alias("review_text"),
)

assert len(result.fetchall()) == 2
con.close()
```

Actor 丢失会导致 query 失败，分布式执行也不定义稳定的全局行求值顺序。不要把这个契约用作持久、全局、keyed 或 transactional 状态。精确限制见 [UDF API 参考](/zh-CN/docs/data/reference/udf-api)，原因与失败模型见 [UDF 概念](/zh-CN/docs/data/concepts/udfs)。如果只是需要复用昂贵的模型或客户端，并且可以接受多个相互独立的 actor，请使用下文的 actor-backed Relation 模式。

## Relation API：只用于表形能力

只有任务需要表形能力或显式 Relation executor 时，才迁移到 Relation API。Vane 不提供这些 UDF 的直接 SQL table-function 形式，因此要在 Python 中构造这些阶段。下面的每个示例都会明确说明必须使用 Relation API 的能力。

### 能力：Arrow 多列或完整表阶段输出 — `rel.map_batches`

这里必须使用 `rel.map_batches`，因为一次调用会同时返回稳定源字段和多个新列。Relation 结果只包含声明的 schema，因此必须显式返回需要透传的字段。

```python
import vane

import re
import duckdb
import pyarrow as pa


def prepare_review_batch(table):
    document_ids = table.column("document_id").to_pylist()
    source_uris = table.column("source_uri").to_pylist()
    texts = table.column("text").to_pylist()
    review_text = [
        None if text is None else re.sub(r"\s+", " ", text).strip().lower()
        for text in texts
    ]
    text_length = [None if text is None else len(text) for text in review_text]
    status = ["missing_text" if text is None else "ok" for text in review_text]
    return pa.table(
        {
            "document_id": pa.array(document_ids, type=pa.string()),
            "source_uri": pa.array(source_uris, type=pa.string()),
            "review_text": pa.array(review_text, type=pa.string()),
            "text_length": pa.array(text_length, type=pa.int64()),
            "status": pa.array(status, type=pa.string()),
        }
    )


con = vane.connect()
source = con.sql(
    """
    SELECT
        document_id::VARCHAR AS document_id,
        source_uri::VARCHAR AS source_uri,
        text::VARCHAR AS text
    FROM (VALUES
        ('doc-001', 's3://sample/doc-001.txt', ' Missing   signature '),
        ('doc-002', 's3://sample/doc-002.txt', NULL)
    ) AS source_rows(document_id, source_uri, text)
    """
)
review_schema = {
    "document_id": duckdb.sqltypes.VARCHAR,
    "source_uri": duckdb.sqltypes.VARCHAR,
    "review_text": duckdb.sqltypes.VARCHAR,
    "text_length": duckdb.sqltypes.BIGINT,
    "status": duckdb.sqltypes.VARCHAR,
}

result = source.map_batches(
    prepare_review_batch,
    schema=review_schema,
    batch_size=2,
    execution_backend="subprocess_task",
)

assert list(result.columns) == list(review_schema)
assert len(result.fetchall()) == 2
con.close()
```

### 能力：显式零行、一行或多行 — `rel.flat_map`

`rel.flat_map` 是 filter 或 expansion 的显式基数边界。此任务针对 null 文本输出零个 token，针对非 null 文本输出多个 token，并把稳定 ID 和源 URI 带入每一行输出。

```python
import vane

import duckdb


def split_document(row):
    text = row["text"]
    if text is None:
        return
    for token_index, token in enumerate(text.split()):
        yield {
            "document_id": row["document_id"],
            "source_uri": row["source_uri"],
            "token_index": token_index,
            "token": token.lower(),
        }


con = vane.connect()
source = con.sql(
    """
    SELECT
        document_id::VARCHAR AS document_id,
        source_uri::VARCHAR AS source_uri,
        text::VARCHAR AS text
    FROM (VALUES
        ('doc-001', 's3://sample/doc-001.txt', 'Alpha Beta'),
        ('doc-002', 's3://sample/doc-002.txt', NULL)
    ) AS source_rows(document_id, source_uri, text)
    """
)
token_schema = {
    "document_id": duckdb.sqltypes.VARCHAR,
    "source_uri": duckdb.sqltypes.VARCHAR,
    "token_index": duckdb.sqltypes.INTEGER,
    "token": duckdb.sqltypes.VARCHAR,
}

tokens = source.flat_map(
    split_document,
    schema=token_schema,
    execution_backend="subprocess_task",
)

assert tokens.order("document_id, token_index").fetchall() == [
    ("doc-001", "s3://sample/doc-001.txt", 0, "alpha"),
    ("doc-001", "s3://sample/doc-001.txt", 1, "beta"),
]
con.close()
```

### 能力：使用显式 Relation Executor 的逐行 Scalar 输出 — `rel.map`

当前 `rel.map` 会把全部输入列作为 positional value 传给 scalar function，并返回原始列以及名为 `value` 的结果列。它不是过时的 pandas DataFrame batch mapping API。当你确实需要标准化的透传列加 `value` 契约，或显式 Relation backend 控制时，再使用它。

```python
import vane

import duckdb


def build_review_key(document_id, source_uri, text):
    text_size = len(text or "")
    return f"{document_id}:{text_size}"


con = vane.connect()
source = con.sql(
    """
    SELECT
        document_id::VARCHAR AS document_id,
        source_uri::VARCHAR AS source_uri,
        text::VARCHAR AS text
    FROM (VALUES
        ('doc-001', 's3://sample/doc-001.txt', 'Alpha'),
        ('doc-002', 's3://sample/doc-002.txt', NULL)
    ) AS source_rows(document_id, source_uri, text)
    """
)

mapped = source.map(
    build_review_key,
    return_type=duckdb.sqltypes.VARCHAR,
    execution_backend="subprocess_task",
)
result = mapped.project(
    "document_id, source_uri, text, value AS review_key"
)

assert result.order("document_id").fetchall() == [
    ("doc-001", "s3://sample/doc-001.txt", "Alpha", "doc-001:5"),
    ("doc-002", "s3://sample/doc-002.txt", None, None),
]
con.close()
```

### 能力：跨 Batch 复用模型或客户端 — Actor-Backed Relation Class

当初始化结果需要由一个或多个 Relation actor 复用时，应传入零参数 callable class 本身，而不是已经创建的 instance。每个 actor 都会创建并复用自己的 instance；多个 actor 提供相互独立的复用与并行度，不定义共享或 keyed state。

```python
import vane

import re
import duckdb
import pyarrow as pa


class ReviewRuleSet:
    def __init__(self):
        self.risk_pattern = re.compile(r"missing|expired", re.IGNORECASE)

    def __call__(self, table):
        document_ids = table.column("document_id").to_pylist()
        source_uris = table.column("source_uri").to_pylist()
        texts = table.column("text").to_pylist()
        needs_review = [
            bool(self.risk_pattern.search(text or ""))
            for text in texts
        ]
        return pa.table(
            {
                "document_id": pa.array(document_ids, type=pa.string()),
                "source_uri": pa.array(source_uris, type=pa.string()),
                "needs_review": pa.array(needs_review, type=pa.bool_()),
            }
        )


con = vane.connect()
source = con.sql(
    """
    SELECT
        document_id::VARCHAR AS document_id,
        source_uri::VARCHAR AS source_uri,
        text::VARCHAR AS text
    FROM (VALUES
        ('doc-001', 's3://sample/doc-001.txt', 'Missing signature'),
        ('doc-002', 's3://sample/doc-002.txt', 'Approved')
    ) AS source_rows(document_id, source_uri, text)
    """
)
review_schema = {
    "document_id": duckdb.sqltypes.VARCHAR,
    "source_uri": duckdb.sqltypes.VARCHAR,
    "needs_review": duckdb.sqltypes.BOOLEAN,
}

result = source.map_batches(
    ReviewRuleSet,
    schema=review_schema,
    batch_size=1,
    execution_backend="subprocess_actor",
    actor_number=2,
    gpus=0.0,
)

assert result.order("document_id").fetchall() == [
    ("doc-001", "s3://sample/doc-001.txt", True),
    ("doc-002", "s3://sample/doc-002.txt", False),
]
con.close()
```

对于真实模型或客户端，请在 `__init__` 中或第一次使用时延迟初始化，让每个 actor 拥有并复用自己的资源。Constructor 应保持零参数；把部署相关配置放入可序列化 wrapper、class default 或 worker 环境。

### 扩展 Backend 与资源

改变执行形态前，先在小型本地样本上验证 callable 与 schema：

1. 无状态 function 使用 `subprocess_task`。
2. 本地 callable-class 复用使用 `subprocess_actor`。
3. 只有分布式执行有帮助时，才把无状态工作迁移到 `ray_task`。
4. 需要分布式复用、actor pool 或 Ray 资源调度时，才把 callable-class 工作迁移到 `ray_actor`。

`actor_number` 是 Relation actor 数量，不是 AI provider 的请求 `concurrency`。在 Ray 上，`gpus` 是每个 task 或 actor 的 scheduler reservation；例如 `actor_number=2` 与 `gpus=1.0` 需要两个 Ray GPU 资源位才能满并行运行。在本地 subprocess backend 上，`gpus` 不会 reservation 或隔离设备：worker 只会继承进程可见的 GPU，非零声明可能产生 warning。每个 worker 都必须能够导入 callable，并访问它的 dependency、model artifact、数据路径和凭据。精确的 backend 与资源限制见 [UDF API 参考](/zh-CN/docs/data/reference/udf-api)；部署任务请参阅[单节点部署](/zh-CN/docs/data/deploy/single-node)、[Ray 集群部署](/zh-CN/docs/data/deploy/ray-cluster)和 [GPU 推理](/zh-CN/docs/data/guides/gpu-inference)。

### 输出、错误与副作用契约

把每个 Relation 结果视为显式的表契约：

- 返回与声明 schema 匹配的列和 Arrow-compatible value。
- 包含下游需要的稳定 ID、源 path 或 URI 和业务 key；Relation stage 不会自动保留这些字段。
- 如果预期的逐行失败需要保持可审查，请增加有类型的 `status` 与 `error` 列；未预期的 batch 或 actor 失败应让 query 失败。
- 增加 `batch_size`、`actor_number` 或数据量之前，先校验 schema、SQL 类型、行数或基数预期、null 和 ID。
- 尽量让 UDF 保持 pure。外部副作用不会随 SQL 事务回滚，也不提供 exactly-once；不可避免的写入应使用稳定 ID 作为 key 并保持幂等。
- 只写入一次通过校验的最终 Relation，不要在 UDF 内部写入。

状态、retry 与副作用背后的原因见 [UDF 概念](/zh-CN/docs/data/concepts/udfs)，精确输出和错误限制见 [UDF API 参考](/zh-CN/docs/data/reference/udf-api)。

## 相关指南

- 使用[结构化转换](/zh-CN/docs/data/guides/structured-transformation)完成 UDF 前后的 SQL 清理、join、去重和校验。
- 当 Python 需要读取 bytes、解码媒体，或记录阶段级 status 与 error 时，使用[多模态 Pipeline](/zh-CN/docs/data/guides/multimodal-pipeline)。
- 标准 prompting、embedding 和 classification 使用 [AI Function](/zh-CN/docs/data/guides/ai-functions)；自定义预处理、模型执行、客户端或输出 schema 使用自定义 UDF。
- 本地输出契约正确后，再使用[性能调优](/zh-CN/docs/data/guides/performance-tuning)。
