---
title: 多模态数据摄取
---

Vane Data 将 multimodal record 视为 table row。Path、bytes、metadata、decoded feature、label、embedding 和 model output 都可以作为显式 column 在同一个 relation 中流转。

当你需要 SQL filtering 和 table output，但 pipeline 的一部分必须调用 Python library 做 decoding、feature extraction 或 model inference 时，使用这个 pattern。

## Ingest Pattern

1. 构建包含稳定 ID、source location、metadata 和可选 bytes 的 relation。
2. 在昂贵的 Python work 前使用 SQL filter 和 project。
3. 使用 `map_batches` 做 decoding、feature extraction 或 model inference。
4. 保持 UDF output schema 显式。
5. 将 curated output 写到 Parquet 或其他 DuckDB-supported target。

Vane Data 不定义通用 multimodal file format。请根据下游系统选择 column name 和 output format。

## Images

对于本地 image file，先从 metadata 和 bytes 开始：

```python
from pathlib import Path

import pyarrow as pa
import vane

rows = []
for idx, path in enumerate(Path("images").glob("*.png")):
    rows.append({
        "id": idx,
        "source_uri": str(path),
        "image_bytes": path.read_bytes(),
    })

con = vane.connect()
rel = con.from_arrow(pa.table({
    "id": [row["id"] for row in rows],
    "source_uri": [row["source_uri"] for row in rows],
    "image_bytes": [row["image_bytes"] for row in rows],
}))
```

使用 batch UDF decode 或 inspect image：

```python
import io

import pyarrow as pa
from PIL import Image

def inspect_images(batch: pa.Table) -> pa.Table:
    ids = batch.column("id").to_pylist()
    source_uris = batch.column("source_uri").to_pylist()
    sizes = []
    modes = []
    for data in batch.column("image_bytes").to_pylist():
        image = Image.open(io.BytesIO(data))
        sizes.append(f"{image.width}x{image.height}")
        modes.append(image.mode)
    return pa.table({
        "id": ids,
        "source_uri": source_uris,
        "image_size": sizes,
        "image_mode": modes,
    })

out = rel.map_batches(
    inspect_images,
    schema={
        "id": "BIGINT",
        "source_uri": "VARCHAR",
        "image_size": "VARCHAR",
        "image_mode": "VARCHAR",
    },
    batch_size=32,
    execution_backend="subprocess_task",
)

out.show()
```

对于 model inference，尽可能把 decoding 和 inference 放在不同 UDF stage 中。这样更容易隔离 failure，也能独立调节 batch size。

## Audio

将 audio 表示为 source URI 加 bytes，或 source URI 加已解析的本地 path。使用 UDF 做 transcription 或 acoustic feature extraction。

推荐 output column：

- `audio_id`
- `source_uri`
- `transcript`
- `language`
- `segment_json`，或每个 segment 一行
- 使用 segment row 时记录 `start_time` 和 `end_time`

将 provider-specific transcription code 放在 UDF 内。Vane Data 管理 relation、batching、schema 和 execution backend；它不提供内置 transcription engine。

## Documents

对于 PDF、office document、HTML 或长文本文件，将 extraction、chunking 和 embedding 保持为独立 stage：

1. Extract text 和 document metadata。
2. 将 text 切成带稳定 ID 的 chunk。
3. 对 chunk 做 embedding 或 classification。
4. 写出 document-level 和 chunk-level output。

推荐 column：

- `document_id`
- `source_uri`
- `page_number`
- `chunk_id`
- `text`
- `embedding`
- 存储 model output 时记录 model/provider metadata

## Video

将 video ingest 视为更重的 pipeline：

1. 读取 metadata 和 file path。
2. 在受控 batch 中 decode frame 或 clip。
3. 对 frame 或 clip 运行 model UDF。
4. 带 timestamp 存储 frame-level 或 clip-level output。

避免从 UDF 返回大型 Python object。请将 output 转成 Arrow-compatible value，例如 string、number、list、bytes 或 structured JSON string。

## Local vs Ray

先从本地开始：

```python
out = rel.map_batches(
    decode_batch,
    schema={"id": "BIGINT", "feature": "VARCHAR"},
    batch_size=32,
    execution_backend="subprocess_task",
)
```

当 input size、decoding cost、model throughput 或 GPU placement 需要时，再迁移到 Ray：

```python
import vane

vane.configure(runner="ray")

out = rel.map_batches(
    ModelBatch,
    schema={"id": "BIGINT", "label": "VARCHAR"},
    batch_size=64,
    execution_backend="ray_actor",
    gpus=1,
    concurrency=4,
)
```

将 `decode_batch` 和 `ModelBatch` 替换为你自己的 function 或 callable class。当后续 stage 需要把 model output join 回 source record 时，请从每个 UDF 返回稳定 ID。每个 Ray worker 都必须能够 import callable，并访问相同 storage、model file 和 credential。

## Practical Checks

- 在 decoding 或 model inference 前保留稳定 ID。
- 存储 source URI 和足够 metadata，以便复现一行数据。
- 可能时，在加载 bytes 前先 filter 和 project。
- 保持 UDF output 小且类型明确。
- 运行大型 media job 前先校验小 sample。
- 使用 Ray 时，将 output 写到所有 worker 都能访问的 filesystem。
