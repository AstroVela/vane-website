---
title: SQL 多模态 Pipeline
---

本指南展示一个 SQL-first 的 multimodal data preparation pattern。SQL 负责 metadata filtering 和 validation；Python UDF 处理 media-specific step；结果再回到 SQL 中做 inspection 和 output。

当 pipeline 的大部分逻辑都应该让 SQL 用户可读，但其中一个 stage 需要 Python library 做 decoding、feature extraction 或 model inference 时，使用这种方式。

## Pipeline Shape

这个 pattern 有五个 stage：

1. 使用 SQL 选择 candidate row。
2. 加载 bytes，或保留稳定 media reference。
3. 运行 Python UDF 做 decoding 或 inference。
4. 回到 SQL 做 validation 和 summary。
5. 写出 curated tabular output。

Vane Data 不会给 SQL 增加 image、audio 或 video syntax。Multimodal work 会被表示为 table column 和显式 Python UDF stage。

## 1. 从 SQL Metadata 开始

第一步保持 relational。先 filter row、normalize metadata，并在 Python 看到数据前减少 input。

```python
import vane

con = vane.connect()

items = con.sql("""
    select
        id,
        path,
        media_type
    from read_parquet('data/media_index/*.parquet')
    where media_type in ('image/png', 'image/jpeg')
      and path is not null
""")
```

## 2. 添加 Bytes 或 Reference

如果 media bytes 已经在 Parquet 中，保留在 relation 里并跳过这一步。如果 table 存储的是 path，使用 UDF 加载 bytes，并返回后续 stage 需要的 column。

```python
from pathlib import Path

import pyarrow as pa

def load_image_bytes(batch: pa.Table) -> pa.Table:
    ids = batch.column("id").to_pylist()
    paths = batch.column("path").to_pylist()
    media_types = batch.column("media_type").to_pylist()

    return pa.table({
        "id": ids,
        "path": paths,
        "media_type": media_types,
        "image_bytes": [Path(path).read_bytes() for path in paths],
    })

with_bytes = items.map_batches(
    load_image_bytes,
    schema={
        "id": "BIGINT",
        "path": "VARCHAR",
        "media_type": "VARCHAR",
        "image_bytes": "BLOB",
    },
    batch_size=32,
    execution_backend="subprocess_task",
)
```

`map_batches` 返回 UDF 产出的列。当 downstream stage 仍然需要 `id`、`path` 和 `media_type` 等 input column 时，请从 UDF 中返回它们。

## 3. 运行 Model Step

当 model、tokenizer、image processor 或 client 应该初始化一次并跨 batch 复用时，使用 actor backend。

```python
class ImageClassifier:
    def __init__(self) -> None:
        # Initialize model state here in a real pipeline.
        self.labels = ("invoice", "claim", "other")

    def __call__(self, batch: pa.Table) -> pa.Table:
        ids = batch.column("id").to_pylist()
        paths = batch.column("path").to_pylist()
        media_types = batch.column("media_type").to_pylist()
        image_bytes = batch.column("image_bytes").to_pylist()

        labels = []
        scores = []
        for value in image_bytes:
            labels.append(self.labels[0] if value else self.labels[2])
            scores.append(1.0 if value else 0.0)

        return pa.table({
            "id": ids,
            "path": paths,
            "media_type": media_types,
            "label": labels,
            "score": scores,
        })

classified = with_bytes.map_batches(
    ImageClassifier,
    schema={
        "id": "BIGINT",
        "path": "VARCHAR",
        "media_type": "VARCHAR",
        "label": "VARCHAR",
        "score": "DOUBLE",
    },
    batch_size=16,
    execution_backend="subprocess_actor",
)
```

这个示例 classifier 有意保持简单，方便看清 relation contract。在生产中，把 decoding、preprocessing 和 inference 放进 actor，同时保持返回 schema 稳定。

## 4. 回到 SQL

当你想用普通 SQL 做 validation、summary 或 downstream join 时，从 relation 创建 table。

```python
classified.to_table("classified_media")

summary = con.sql("""
    select
        label,
        count(*) as rows,
        avg(score) as avg_score
    from classified_media
    group by label
    order by rows desc
""")

summary.show()
```

## 5. 写出 Output

只写出 downstream system 需要的 column。Large media bytes 在 inference 期间通常有用，但在 curated output 中往往不需要。

```python
curated = con.sql("""
    select id, path, media_type, label, score
    from classified_media
""")

curated.write_parquet("output/classified_media/")
```

## 迁移到 Ray

当 expensive stage 迁移到 Ray 时，SQL text 和 UDF contract 不需要变化。请在创建并执行 pipeline 前配置 runner：

```python
import vane

vane.configure(runner="ray")
```

然后为 expensive stage 选择 Ray UDF backend：

```python
classified = with_bytes.map_batches(
    ImageClassifier,
    schema={
        "id": "BIGINT",
        "path": "VARCHAR",
        "media_type": "VARCHAR",
        "label": "VARCHAR",
        "score": "DOUBLE",
    },
    batch_size=16,
    execution_backend="ray_actor",
    gpus=1,
    concurrency=4,
)
```

每个 Ray worker 都必须能够 import UDF class，并访问相同 storage、credential、model file 和 Python dependency。

## Practical Checks

在推广 multimodal pipeline 前：

- 使用小型本地 run 验证 UDF output schema。
- 将 filter 和 column projection 放在 byte loading 和 inference 之前。
- 保留稳定 identifier，使 model output 可以 join 回 source record。
- 决定 media bytes 属于 final output，还是只属于 intermediate stage。
- 扩展规模前，先从每个 Ray worker 测试 path、storage 和 credential access。
