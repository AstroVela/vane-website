---
title: 性能调优
---

先让本地 pipeline 正确，再调节最少数量的变量。

## Runner

Local：

```python
vane.configure(runner="native")
```

Ray：

```python
vane.configure(runner="ray")
```

Native runner 会流式处理 Arrow record batch。Ray runner 会在需要时初始化 Ray，并将 logical plan 发送给 Ray query driver。

## Batch Size

对于 `map_batches`，`batch_size` 控制每次 UDF call 传入多少行。

小 batch：

- memory 更低
- model throughput 更差
- scheduling overhead 更多

大 batch：

- throughput 更好
- memory 更高
- retry 更慢

请使用 representative data 调优。

## Actor Reuse

model inference 使用 actor backend：

- `subprocess_actor` 用于本地 stateful reuse。
- `ray_actor` 用于 distributed stateful reuse。

轻量 stateless work 使用 task backend：

- `subprocess_task`
- `ray_task`

## CPU 和 GPU Resource

`map_batches` 接受：

- `cpus`
- `gpus`
- `concurrency`

Ray 使用 resource request 做 placement。增大 `concurrency` 前，请确认 Ray cluster 暴露了匹配的 resource。

## Ray Scan Task Grouping

Public config：

```python
vane.configure(ray_scan_task_size_grouping=True)
```

环境变量：

```bash
export VANE_RAY_SCAN_TASK_SIZE_GROUPING=true
```

Registry 将它描述为基于 size 的 scan task grouping，适合将小文件合并成更大的 scan task。

相关 public config：

- `ray_max_task_backlog`
- `ray_scan_task_open_cost_bytes`
- `ray_scan_task_min_partition_num`
- `ray_init_sql`

## Backpressure

当 `VANE_RAY_MAX_TASK_BACKLOG` 设置为正值时，它会限制 pending Ray task 数量。Public registry 中 `0` 表示 unlimited。

当 driver 提交速度快于 worker 消费速度时使用它。

## Native Runner Batch Size

当你需要为 measurement 或 memory control 指定明确值时，`VANE_NATIVE_RUNNER_BATCH_SIZE` 会覆盖 native runner batch size。

```bash
export VANE_NATIVE_RUNNER_BATCH_SIZE=4096
```

## 避免常见瓶颈

- 只 project UDF 需要的 column。
- 在 model call 前 filter。
- 调优时将 image/audio/video decoding 与 model inference 分离。
- 写入所有 worker 都能访问的 filesystem。
- 避免 model workload 每行一次 UDF call。
- 使用 structured output schema，而不是 Python object column。

## Measure

保留简单的运行记录：

- input rows
- input bytes
- batch size
- backend
- concurrency
- GPUs per actor
- elapsed time
- output rows
- failure count

Benchmark workload 可以作为 environment-driven performance experiment 的参考，但 benchmark-specific file location、storage endpoint 和 model default 不是通用 deployment default。
