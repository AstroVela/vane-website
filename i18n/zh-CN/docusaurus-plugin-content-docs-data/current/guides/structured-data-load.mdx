---
title: 结构化数据加载
---

当你的 AI 或 multimodal workflow 从 table-shaped input 开始时，使用本指南：media manifest、document metadata、text record、label、prompt table、model output、evaluation sample，或需要与 source object join 的 business ID。

## 加载本地 Parquet

对 Parquet 文件和 Parquet part 目录使用 `read_parquet(...)`。目录通常通过指向其内部文件的 glob 读取。

```python
import vane

con = vane.connect()

documents = con.sql("""
    select *
    from read_parquet('data/document_manifest/*.parquet', union_by_name=true)
    where text is not null
    order by document_id
    limit 10000
""")

documents.to_table("raw_documents")
```

对于嵌套目录，使用 recursive glob：

```python
media_events = con.sql("""
    select *
    from read_parquet('data/media_events/**/*.parquet', union_by_name=true)
    limit 10000
""")
```

当多个文件的列顺序可能不同，或预期存在少量 schema drift 时，使用 `union_by_name=true`。如果 schema 的含义不同，请先停止并完成 reconcile，再继续。

## 加载本地 CSV

CSV 适合 label、review sample、prompt table 或 issue list 等小型导出。一旦你能控制导出格式，优先使用 Parquet，因为它能更可靠地携带 column type。

```python
labels = con.sql("""
    select *
    from read_csv_auto('data/labels/*.csv', header=true)
    limit 10000
""")

labels.to_table("raw_labels")
```

如果类型推断错误，保持加载逻辑简单，先检查 inferred schema，再在 source relation 中显式 cast 重要列。

## 从 S3 或 MinIO 加载


```python
import vane

con = vane.connect()
con.execute("LOAD httpfs")

# Prefer environment variables for credentials:
# AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY, AWS_SESSION_TOKEN when needed.
con.execute("SET s3_region='us-east-1'")
con.execute("SET s3_url_style='path'")

source = con.sql("""
    select *
    from read_parquet('s3://my-bucket/sample/document_manifest/*.parquet')
    limit 10000
""")
```

对于 MinIO 或其他 S3-compatible endpoint，添加 endpoint setting：

```python
con.execute("SET s3_endpoint='localhost:9000'")
con.execute("SET s3_use_ssl=false")
con.execute("SET s3_url_style='path'")
```
