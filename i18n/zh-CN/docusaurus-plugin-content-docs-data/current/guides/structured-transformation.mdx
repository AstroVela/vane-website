当你希望在一个 pipeline 中完成 SQL filtering、Python text processing 和 table output 时，使用 Vane Data 做 deduplication 和 cleaning。

Vane Data 不提供内置 MinHash 或 data-quality framework。它提供 relation management、SQL、UDF execution 和 write path；cleaning 与 deduplication logic 仍然是显式 Python 或 SQL code。

## Pipeline Shape

典型 text-cleaning pipeline 如下：

1. 加载带稳定 ID 和 source metadata 的 text row。
2. 使用 SQL 或 Python UDF normalize text。
3. 当 normalized text 完全相同时，移除 exact duplicate。
4. 只有当 exact dedup 不够时，才使用 MinHash 等 approximate similarity logic。
5. 每个 duplicate group 保留一个 representative。
6. 写出 retained row 和 audit column。

## Minimal Cleaning UDF

```python
import re

import pyarrow as pa

def clean_text(batch: pa.Table) -> pa.Table:
    ids = batch.column("id").to_pylist()
    text = batch.column("text").to_pylist()
    cleaned = []

    for value in text:
        s = str(value or "")
        s = re.sub(r"\s+", " ", s).strip().lower()
        cleaned.append(s)

    return pa.table({
        "id": ids,
        "text": text,
        "clean_text": cleaned,
    })

cleaned = rel.map_batches(
    clean_text,
    schema={
        "id": "BIGINT",
        "text": "VARCHAR",
        "clean_text": "VARCHAR",
    },
    batch_size=1024,
    execution_backend="subprocess_task",
)
```

在 validation 完成前保留 original text column。Cleaning rule 如果过于激进，可能会移除有用信息。

## 使用 SQL 做 Exact Dedup

对于小型或高度 normalized 的 dataset，SQL exact dedup 通常已经足够：

```python
cleaned.to_table("cleaned")

deduped = con.sql("""
    select
        any_value(id) as id,
        any_value(text) as text,
        clean_text,
        count(*) as duplicate_count
    from cleaned
    group by clean_text
""")
```

这会为每个 normalized text value 保留一个 representative，并记录每个 group 折叠了多少行。

## Near-Duplicate Strategy

对于 near-duplicate text，使用 Python UDF 计算 signature，再使用 SQL 做 grouping 和 scoring。

常见流程：

1. Normalize text。
2. 在 `map_batches` 中计算 signature，例如 MinHash。
3. 用 bounded LSH bucket 生成 candidate pair。
4. Score candidate。
5. 构建 duplicate component。
6. 每个 component 保留一个 representative。

保持 candidate expansion 有边界。当 bucket 过大时，approximate dedup pipeline 可能会变得昂贵。

## 需要保留的 Column

对于 training data 和 retrieval dataset，保留 audit-friendly column：

- stable source ID
- original URI 或 file path
- raw text
- normalized text
- duplicate component ID
- representative flag
- duplicate count 或 similarity score
- filter reason 或 quality score

这些 column 让你能解释一行为什么被保留或移除。

## Scaling

从本地 execution 开始：

```python
signatures = cleaned.map_batches(
    signature_batch,
    schema={
        "id": "BIGINT",
        "clean_text": "VARCHAR",
        "signature": "UBIGINT[]",
    },
    batch_size=1024,
    execution_backend="subprocess_task",
)
```

当 distribution 有帮助时，将 CPU-heavy signature generation 迁移到 Ray：

```python
import vane

vane.configure(runner="ray")

signatures = cleaned.map_batches(
    signature_batch,
    schema={
        "id": "BIGINT",
        "clean_text": "VARCHAR",
        "signature": "UBIGINT[]",
    },
    batch_size=2048,
    execution_backend="ray_task",
)
```

将 `signature_batch` 替换为你自己的 deterministic signature function。请返回 stable ID 和 normalized text，以及 signature，这样后续 SQL stage 可以 audit 和 group row。

## Practical Checks

- 信任规则前，先抽样检查 removed row。
- 跟踪每个 stage 的 row count。
- 保留 raw text，直到 final validation step。
- 让 normalization deterministic。
- 让 representative selection deterministic。
- 当 audit 很重要时，分别写出 retained row 和 rejected-row summary。
