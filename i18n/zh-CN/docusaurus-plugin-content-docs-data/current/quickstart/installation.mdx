---
title: 安装
---

Vane Data 使用 `vane-ai` 作为发布包名，使用 `vane` 作为 Python 导入包。

## 环境要求

- Python 3.10 或更高版本。
- 平台已有可用的 Vane Data wheel，或本地具备用于源码构建的 C++ 构建环境。
- pipeline 使用的远程存储、模型 provider 或 Ray 集群所需的网络和凭证。

包元数据会安装必要的运行时依赖，包括 `cloudpickle`、`numpy`、`pyarrow` 和 `ray`。

## 安装发布包

当你的 Python 版本和平台已有可用 wheel 时，优先安装发布包：

```bash
pip install vane-ai
```

AI provider adapter 会按需加载。安装 `vane-ai` 后，只需再添加实际使用的 provider 库：

```bash
# Local embedding and classification models
python -m pip install sentence-transformers transformers torch

# Hosted model providers
python -m pip install openai
python -m pip install anthropic
python -m pip install google-genai

# vLLM-backed prompting
python -m pip install vllm
```

## 验证安装

```bash
python - <<'PY'
import vane

print("Vane:", vane.__version__)
print("DuckDB:", vane.__duckdb_version__)

con = vane.connect()
con.sql("select 42 as answer").show()
PY
```

## 默认 Ray 执行

Ray 是 Vane 的默认 runner。保持 `RAY_ADDRESS` 未设置时，上面的验证命令会自动启动本机 Ray runtime，因此不需要显式配置 runner。

如果进程需要脱离 Ray 运行，请在创建 connection 前选择 `local`：

```python
import vane

vane.configure(runner="local")

con = vane.connect()
con.sql("select 42 as answer").show()
```

也可以在启动进程时做相同选择：

```bash
VANE_RUNNER=local python job.py
```

要使用已有 Ray 集群，请在启动作业前设置 `RAY_ADDRESS`。每个 Ray worker 都必须能够导入相同的 Python 包、访问相同的存储系统，并能访问 pipeline 所需的模型文件或 provider 凭证。本机和集群 runner 设置请参阅[部署](/zh-CN/docs/data/deploy/deployment)。

## 从源码构建

当你正在开发 Vane Data、测试尚未发布的改动，或使用的平台没有匹配的 wheel 时，可以从源码构建。

克隆代码库，并拉取子模块：

```bash
git clone --recursive https://github.com/AstroVela/vane.git
cd vane
```

如果检出目录已存在但子模块缺失：

```bash
git submodule update --init --recursive
```

安装常见 Debian 或 Ubuntu 构建工具：

```bash
sudo apt-get update
sudo apt-get install -y build-essential cmake ninja-build pkg-config curl zip unzip tar flex bison
```

使用代码库锁定的 vcpkg 版本构建 C++ 依赖：

```bash
git clone https://github.com/microsoft/vcpkg.git .cache/vcpkg
git -C .cache/vcpkg checkout 44819aa2a6c10e56065e2b0330e7d6c89d1d2574
.cache/vcpkg/bootstrap-vcpkg.sh -disableMetrics
.cache/vcpkg/vcpkg install --x-install-root="$PWD/vcpkg_installed"
```

安装 Python 构建工具，然后构建并安装非 editable 的 Release wheel：

```bash
python -m pip install -U pip
python -m pip install cmake ninja scikit-build-core "pybind11[global]"

export SKBUILD_BUILD_DIR="$PWD/build/release"
export SKBUILD_CMAKE_BUILD_TYPE=Release

python -m pip install . --no-build-isolation -v
```

`vcpkg install` 会读取代码库中的 `vcpkg.json` manifest，并把构建好的依赖写入 `vcpkg_installed`。Vane 的 CMake 配置会自动发现该目录，因此不要设置 `CMAKE_TOOLCHAIN_FILE`。代码库已经忽略 `.cache` 和 `vcpkg_installed`，它们不会被 Git 提交。

这里有意使用非 editable 安装：默认 Ray runner 会在 worker 进程中导入 Vane，而 scikit-build-core 的 editable rebuild mode 可能在每个 worker 导入时再次调用 CMake。修改源码后，请重新运行安装命令。如果现有检出目录使用过不同的 CMake 配置，请选择新的 `SKBUILD_BUILD_DIR`，或在重新构建前删除对应的旧构建目录。

## 构建配置说明

当前源码构建配置包含以下 DuckDB 扩展：

```text
core_functions;json;parquet;icu;jemalloc;httpfs
```

这意味着 JSON、Parquet、ICU、jemalloc，以及 HTTP/S3 文件系统支持都属于当前配置构建的一部分。其他 DuckDB 扩展取决于构建和运行环境。请在作业实际运行的环境中验证扩展可用性。

项目包含面向 Linux manylinux 2.28（x86_64 和 aarch64）、macOS 以及 Windows 的 wheel 构建配置。实际包可用性取决于对应版本提供的发布产物。

## 故障排查

- `ModuleNotFoundError: No module named 'vane'`：确认 `vane-ai` 已安装到运行脚本的 Python 环境中。
- 缺少 provider 库：直接安装对应 provider 包，例如 `openai`、`anthropic`、`google-genai`、`vllm` 或 `sentence-transformers`。
- Ray worker 无法导入某个模块：在每个 worker 上安装相同依赖，或提供包含这些依赖的 Ray 运行时环境。
- 远程文件读取失败：检查 DuckDB 扩展、存储凭证、端点设置，以及 worker 侧环境变量。
