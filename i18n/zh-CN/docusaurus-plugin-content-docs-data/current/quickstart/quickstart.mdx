这篇快速开始会让一个 Python 函数在 Vane SQL projection 中运行起来。必需路径使用内联数据和本地执行，不需要 Ray 集群、GPU、PyArrow 或模型 provider 凭据。

你将创建三行数据，把一个 Python 函数注册为 SQL alias，在 `SELECT` 中调用它，并看到结果。

## 安装 Vane Data

```bash
python -m pip install vane-ai
```

## 运行第一个 SQL Expression UDF

把下面的内容复制到一个 Python 文件并运行：

```python
import vane

con = vane.connect()

con.execute("""
    CREATE TABLE documents AS
    SELECT *
    FROM (VALUES
        ('doc-001', 'Signed claim form received.'),
        ('doc-002', 'Invoice is missing approval metadata.'),
        ('doc-003', 'Policy reference has expired.')
    ) AS rows(document_id, source_text)
""")


def review_status(source_text):
    text = str(source_text).lower()
    if "missing" in text or "expired" in text:
        return "needs_review"
    return "ready"


vane.attach_function(
    review_status,
    alias="review_status_sql",
    connection=con,
    parameters=["VARCHAR"],
    return_dtype="VARCHAR",
)

reviewed = con.sql("""
    SELECT
        document_id,
        source_text,
        review_status_sql(source_text) AS review_status
    FROM documents
    ORDER BY document_id
""")

reviewed.show()
vane.detach_function("review_status_sql", connection=con)
```

结果会保留原始列，并新增一个有类型的 `review_status` 列：

| document_id | review_status |
| --- | --- |
| doc-001 | ready |
| doc-002 | needs_review |
| doc-003 | needs_review |

## 刚才发生了什么

- `vane.attach_function(...)` 把原始 Python callable 注册到 `con`。
- `review_status_sql(...)` 成为可在 `SELECT` projection 中运行的 SQL Expression。
- Expression 为每一输入行增加一个结果，因此 ID 和源文本仍在同一个 relation 中。
- `show()` 物化惰性结果后，`vane.detach_function(...)` 移除了 alias。

异常安全的清理、batch function、callable class 和 Relation 转换请继续阅读[自定义 Python UDF](/zh-CN/docs/data/guides/custom-python-udfs)。精确签名和限制请查阅 [UDF API 参考](/zh-CN/docs/data/reference/udf-api)。

## 可选：添加 AI Function

这不是 Quickstart 的必需步骤。如果要试用托管 provider，请安装对应 package，并把凭据设置到 worker 环境中：

```bash
python -m pip install openai
export OPENAI_API_KEY="<your-key>"
```

然后使用 SQL Expression function `ai_prompt` 增加一个 AI 结果：

```python
with_ai = con.sql("""
    SELECT
        document_id,
        ai_prompt(
            source_text,
            struct_pack(
                provider := 'openai',
                model := 'gpt-4o-mini'
            )
        ) AS review_note
    FROM documents
    ORDER BY document_id
""")

with_ai.show()
```

Provider 调用可能产生费用，并把数据发送到你的环境之外。第一次运行时请保持输入较小；构建更大的 workflow 前，请阅读 [AI Function](/zh-CN/docs/data/guides/ai-functions)。

## 从 Local 扩展到 Ray

Quickstart 默认使用 local runner。要分布式执行同样的 UDF 或 AI Expression，请在创建连接**之前**配置 Ray：

```python
import vane

vane.configure(runner="ray")
con = vane.connect()
```

SQL projection 和注册的 callable 可以保持不变。处理更大任务时，把内联 `VALUES` 表替换为 Parquet 或对象存储等共享数据源。

每个 Ray worker 都必须能够导入 UDF 及其依赖，并访问相同的数据、模型文件和工作所需凭据。在集群上运行前，请阅读 [Ray 集群部署](/zh-CN/docs/data/deploy/ray-cluster)。
