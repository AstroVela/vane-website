这篇快速开始会让一个 Python 函数在 Vane SQL projection 中运行起来。必需路径使用内联数据并在本机运行，不需要外部 Ray 集群、GPU 或模型 provider 凭据。

你将创建三行数据，把一个 Python 函数注册为 SQL alias，在 `SELECT` 中调用它，并看到结果。

开始前，请先完成[安装指南](/zh-CN/docs/data/quickstart/installation)。

## 运行第一个 SQL Expression UDF

把下面的内容复制到一个 Python 文件并运行：

```python
import vane

con = vane.connect()

documents = con.values(
    (
        vane.lit("doc-001").alias("document_id"),
        vane.lit("Signed claim form received.").alias("source_text"),
    ),
    (vane.lit("doc-002"), vane.lit("Invoice is missing approval metadata.")),
    (vane.lit("doc-003"), vane.lit("Policy reference has expired.")),
)


def review_status(source_text):
    text = str(source_text).lower()
    if "missing" in text or "expired" in text:
        return "needs_review"
    return "ready"


vane.attach_function(
    review_status,
    alias="review_status_sql",
    connection=con,
    parameters=["VARCHAR"],
    return_dtype="VARCHAR",
)

reviewed = con.sql("""
    SELECT
        document_id,
        source_text,
        review_status_sql(source_text) AS review_status
    FROM documents
    ORDER BY document_id
""")

reviewed.show()
vane.detach_function("review_status_sql", connection=con)
```

结果会保留原始列，并新增一个有类型的 `review_status` 列：

| document_id | review_status |
| --- | --- |
| doc-001 | ready |
| doc-002 | needs_review |
| doc-003 | needs_review |

## 刚才发生了什么

- `vane.attach_function(...)` 把原始 Python callable 注册到 `con`。
- `review_status_sql(...)` 成为可在 `SELECT` projection 中运行的 SQL Expression。
- Expression 为每一输入行增加一个结果，因此 ID 和源文本仍在同一个 relation 中。
- `show()` 物化惰性结果后，`vane.detach_function(...)` 移除了 alias。

Callable 生命周期、batch 与基数行为以及 Relation 转换请继续阅读 [UDF 概念](/zh-CN/docs/data/concepts/udfs)。

## 可选：添加 AI Function

这不是 Quickstart 的必需步骤。如果要试用 OpenAI，请在同一个环境中安装它的可选依赖，然后把凭据设置到 worker 环境中：

```bash
uv pip install openai
export OPENAI_API_KEY="<your-key>"
```

然后使用 SQL Expression function `ai_prompt` 增加一个 AI 结果：

```python
with_ai = con.sql("""
    SELECT
        document_id,
        ai_prompt(
            source_text,
            struct_pack(
                provider := 'openai',
                -- OpenAI-compatible provider 可按需配置：
                -- base_url := 'https://api.example.com/v1',
                model := 'gpt-4o-mini'
            )
        ) AS review_note
    FROM documents
    ORDER BY document_id
""")

with_ai.show()
```

Provider 调用可能产生费用，并把数据发送到你的环境之外。第一次运行时请保持输入较小；构建更大的 workflow 前，请阅读 [AI Function 概念](/zh-CN/docs/data/concepts/ai-functions)。

## 从本机 Ray 扩展到集群

Ray 是 Vane 的默认 runner。保持 `RAY_ADDRESS` 未设置时，这篇快速开始会在当前机器启动 Ray runtime，不需要外部集群，也不需要显式配置 runner。

要在集群上运行同一作业，请在 head 节点启用集群自动发现后启动它：

```bash
RAY_ADDRESS=auto python quickstart.py
```

SQL projection 和注册的 callable 可以保持不变。处理更大任务时，把内联 relation 替换为 Parquet 或对象存储等共享数据源。

每个 Ray worker 都必须能够导入 UDF 及其依赖，并访问相同的数据、模型文件和工作所需凭据。在集群上运行前，请阅读[部署](/zh-CN/docs/data/deploy/deployment)。
