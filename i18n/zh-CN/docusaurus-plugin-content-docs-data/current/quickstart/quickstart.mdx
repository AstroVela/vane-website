这篇 Quickstart 通过一个小型多模态 AI data pipeline 展示 Vane Data：把文档、图像和音频表示为行，使用 SQL 选择有代表性的切片，为审核字段添加 Python 逻辑，可选调用 AI function，然后校验并写出 tabular result。

必需路径使用 inline data 在本地运行。完成必需步骤不需要 Ray、GPU、remote storage、真实媒体文件或 model provider credential。AI 调用是可选步骤，并已明确标注。

## 开始之前

安装 Vane Data，并确认 `import vane` 可以正常工作：

```bash
python -m pip install vane-ai
```

这篇 Quickstart 还会用 PyArrow 作为 Python UDF 的返回值类型：

```bash
python -m pip install pyarrow
```

如果你想运行可选的 hosted AI 步骤，请安装对应 provider package，并设置该 provider 所需的 credential：

```bash
python -m pip install openai
```

## 1. 创建 connection

```python
import pyarrow as pa
import vane

con = vane.connect()
```

运行一个最小 SQL query，确认 connection 可用：

```python
con.sql("select 42 as answer").show()
```

## 2. 创建多模态 relation

第一次运行时使用 inline manifest。它具备真实多模态 workflow 的基本形态：稳定 ID、source location、media type、文本或抽取文本，以及业务 metadata。

```python
assets = con.sql("""
    select *
    from (
        values
            (
                'asset-001',
                's3://sample-bucket/claims/claim-001.pdf',
                'application/pdf',
                'claim',
                'high',
                'Customer submitted a signed claim form with supporting notes.'
            ),
            (
                'asset-002',
                's3://sample-bucket/invoices/invoice-018.png',
                'image/png',
                'invoice',
                'high',
                'Invoice image is missing approval metadata.'
            ),
            (
                'asset-003',
                's3://sample-bucket/calls/call-077.wav',
                'audio/wav',
                'support_call',
                'normal',
                'Transcript mentions an expired policy reference.'
            ),
            (
                'asset-004',
                's3://sample-bucket/memos/memo-009.txt',
                'text/plain',
                'memo',
                'low',
                'Internal follow-up note.'
            )
    ) as t(asset_id, source_uri, media_type, document_type, priority, text)
""")

assets.show()
```

relation 表示表格形态的工作单元。在你的数据中，`source_uri` 可能指向 PDF、图像、音频、视频或抽取文本。关键是每个 asset 都有稳定的行，可以被 filter、enrich、validate，并最终写回输出。

## 3. 使用 SQL 选择 candidate slice

在进入 Python、AI 或模型处理之前，先使用 SQL。可以按 media type、business priority、document type、date range 或任何 metadata 过滤，让第一次运行保持小而有意义。

```python
assets.to_table("assets")

candidates = con.sql("""
    select
        asset_id,
        source_uri,
        media_type,
        document_type,
        priority,
        text
    from assets
    where priority in ('high', 'normal')
      and media_type in ('application/pdf', 'image/png', 'audio/wav')
      and text is not null
    order by asset_id
""")

candidates.show()
```

这个阶段体现了 Vane 面向表的多模态模型：媒体类型可以不同，但 candidate slice 仍然是带显式 column 的 relation。

## 4. 为 AI-ready 字段添加 Python 逻辑

当自定义 Python 代码需要处理 Arrow batch 并返回显式 output column 时，使用 `map_batches(...)`。这个示例会准备 review metadata，以及可发送给 AI function 的 prompt column。

```python
def prepare_review_rows(batch: pa.Table) -> pa.Table:
    asset_ids = batch.column("asset_id").to_pylist()
    source_uris = batch.column("source_uri").to_pylist()
    media_types = batch.column("media_type").to_pylist()
    document_types = batch.column("document_type").to_pylist()
    priorities = batch.column("priority").to_pylist()
    texts = batch.column("text").to_pylist()

    review_priority = []
    status = []
    prompts = []

    for media_type, document_type, priority, text in zip(
        media_types,
        document_types,
        priorities,
        texts,
    ):
        normalized = str(text or "").strip()
        lower_text = normalized.lower()

        if not normalized:
            status.append("missing_text")
            review_priority.append("skip")
            prompts.append("")
            continue

        status.append("ready")
        if priority == "high" or "missing" in lower_text or "expired" in lower_text:
            review_priority.append("high")
        else:
            review_priority.append("normal")

        prompts.append(
            "Review this "
            + document_type
            + " asset of type "
            + media_type
            + ". Return one short business note: "
            + normalized
        )

    return pa.table({
        "asset_id": asset_ids,
        "source_uri": source_uris,
        "media_type": media_types,
        "document_type": document_types,
        "priority": priorities,
        "text": texts,
        "text_length": [len(str(value or "")) for value in texts],
        "review_priority": review_priority,
        "status": status,
        "prompt": prompts,
    })

review_rows = candidates.map_batches(
    prepare_review_rows,
    schema={
        "asset_id": "VARCHAR",
        "source_uri": "VARCHAR",
        "media_type": "VARCHAR",
        "document_type": "VARCHAR",
        "priority": "VARCHAR",
        "text": "VARCHAR",
        "text_length": "BIGINT",
        "review_priority": "VARCHAR",
        "status": "VARCHAR",
        "prompt": "VARCHAR",
    },
    batch_size=1024,
    execution_backend="subprocess_task",
)

review_rows.show()
```

`map_batches` 返回 UDF 的 output column。因此如果后续阶段仍然需要 `asset_id`、`source_uri`、`media_type` 等 identifier 和 source metadata，需要把它们一起返回。

## 5. 可选：调用 AI function

必需的 Quickstart 路径已经展示了多模态 relation 形态和 AI-ready prompt column。如果你已经配置 provider credential，可以使用 `prompt(...)` 运行一个小规模 hosted AI 调用。

```python
ai_only = review_rows.prompt(
    "prompt",
    provider="openai",
    model="gpt-4o-mini",
    system_message="You review business assets. Return one concise note.",
    output_column="ai_review_note",
    batch_size=4,
    execution_backend="subprocess_actor",
)
```

AI helper 会返回配置好的 output column。如果最终输出需要 ID 和 source field，请先检查 row count，再把 helper output 与 source row 合并：

```python
review_table = review_rows.to_arrow_table()
ai_table = ai_only.to_arrow_table()

if ai_table.num_rows != review_table.num_rows:
    raise ValueError("AI output row count does not match source row count")

reviewed_with_ai = con.from_arrow(
    review_table.append_column("ai_review_note", ai_table["ai_review_note"])
)
```

第一次在自己的数据上运行时，先处理 10-100 行，再增加 `batch_size`、`concurrency`，或把 AI stage 迁移到 Ray。

## 6. 校验 tabular result

写出之前，再次使用 SQL 检查输出。

```python
review_rows.to_table("review_rows")

summary = con.sql("""
    select
        media_type,
        review_priority,
        status,
        count(*) as rows,
        avg(text_length) as avg_text_length
    from review_rows
    group by media_type, review_priority, status
    order by media_type, review_priority, status
""")

summary.show()
```

迁移到自己的数据之前，检查你在真实 workflow 中同样会检查的事项：

- row count 符合预期。
- 稳定 ID 仍然存在。
- source URI 和 media type 仍然存在。
- 应该跳过的行具有显式 `status`。
- prompt text 只包含模型应该看到的字段。
- 若干 sample row 在业务上说得通。

## 7. 写出结果

当结果可以进入 review 或 handoff 时，写出 tabular output。

```python
review_rows.write_parquet("quickstart-multimodal-review/")
```

如果运行了可选 AI 步骤，则改为写出 `reviewed_with_ai`：

```python
reviewed_with_ai.write_parquet("quickstart-multimodal-ai-review/")
```

处理自己的数据时，写到按 run 区分的路径，并保留足够的 source metadata，以便复现或解释每一行输出。

## 在你的数据上复用相同形态

本页使用 inline manifest 是为了去掉 setup 噪音。你应该带入自己 workflow 的是以下形态：

```text
load a small multimodal slice -> SQL select candidates -> prepare AI-ready fields -> optional AI function -> SQL validation -> write tabular output
```

先从本地开始。只有在 local data contract、ID、prompt、output schema 和 validation check 都正确之后，再迁移到 Ray、vLLM、GPU inference 或更大规模的 media decoding。
