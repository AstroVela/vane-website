本参考用于查询公开 AI Function 的签名、参数、返回值和调用限制。完整任务请参阅 [AI Function 指南](/zh-CN/docs/data/guides/ai-functions)；执行语义和设计原因请参阅 [AI Function 概念](/zh-CN/docs/data/concepts/ai-functions)。

Vane 有两种 AI API 模型。Expression API 生成一个有类型的 projection 列：推荐使用 SQL 入口，本页随后介绍等价的 Python 入口。Relation API 是专用于内置分块或分类、结构化或多模态 prompt，以及显式 Relation 执行控制的 Python 模型。Vane 没有为这些 Relation helper 提供直接的 SQL table-function 入口。

## Expression API

Expression AI 调用会生成一个保持行数的输出列，可以在 projection 中与稳定 ID 和源列组合。

### SQL 入口（推荐）

在 SQL projection 中直接使用 `ai_prompt` 和 `ai_embed`。它们的可选配置是 query constant，不是逐行数据。

constant `STRUCT` 会先识别下列 framework 字段，再转发其他 provider-specific 扩展：

| Option 组 | 字段 |
| --- | --- |
| 共享 routing 与 execution | `provider`、`model`、`concurrency` / `actor_number`、`batch_size`、`max_retries`、`max_api_concurrency`、`num_gpus` / `gpus_per_actor`、`on_error` |
| Prompt | `system_message` |
| Embedding | `dimensions`、`normalize` |
| vLLM JSON bridge | `engine_args_json`、`generate_args_json` |

这不是封闭 allowlist：其他字段可能继续转发给所选 provider。SQL `Decimal` 值会递归归一化。已知整数字段转为 Python `int` 并拒绝小数值；已知浮点字段转为 `float`；其他整值转为 `int`，小数值转为 `float`。non-finite `Decimal` 会被拒绝。

#### `ai_prompt`

**用途**

在 SQL projection 中为每个输入 message 生成一个文本响应。

**签名**

```sql
ai_prompt(messages VARCHAR [, options CONSTANT]) -> VARCHAR
```

**参数**

`ai_prompt` 接受 1 或 2 个参数。

| 参数 | 契约 |
| --- | --- |
| `messages` | 必需的 `VARCHAR` expression，包含 prompt 或 message 文本。 |
| `options` | 可选的 foldable constant，通常是用 `struct_pack` 构造的 constant `STRUCT`。可以选择 `provider`、`model`、actor/batch/retry 控制、`system_message`、request option 和 provider-specific 配置。 |

**返回值**

每个输入行返回一个 `VARCHAR` 响应。

**行为与数据契约**

该调用是保持行数的 Expression。Options 只绑定一次并转换为 Python 配置 dictionary，不能逐行变化；已识别的 provider 和 request 配置会转发给所选 provider。输出可以在同一 projection 中与 ID 和源列组合。

**最小示例**

```python
import vane

con = vane.connect()
con.execute(
    """
    CREATE TEMP TABLE prompt_source AS
    SELECT *
    FROM (VALUES
        (1, 'Summarize the invoice exception.'),
        (2, 'Summarize the delivery exception.')
    ) AS source(document_id, messages)
    """
)

result = con.sql(
    """
    SELECT
        document_id,
        messages,
        ai_prompt(
            messages,
            struct_pack(
                provider := 'openai',
                model := 'gpt-4o-mini',
                temperature := 0.2
            )
        ) AS response
    FROM prompt_source
    """
)
```

**限制与错误**

第一个参数必须绑定为 `VARCHAR`；第二个参数必须可 fold。该函数只能作为 `SELECT projection` 中的顶层值，不能用在 predicate、grouping、join、aggregate argument 或 short-circuit expression 中。SQL AI 会递归拒绝类似凭据的字段，包括嵌套 key 和 JSON option bridge。凭据必须通过 `OPENAI_API_KEY` 等 worker 环境变量提供，绝不能放入 SQL options。Provider 查找、不受支持的能力和 provider 执行失败会在 expression 规划或求值时报错。

#### `ai_embed`

**用途**

在 SQL projection 中为每个输入文本生成一个 embedding vector。

**签名**

```sql
ai_embed(text VARCHAR [, options CONSTANT]) -> FLOAT[] | FLOAT[N]
```

**参数**

`ai_embed` 接受 1 或 2 个参数。

| 参数 | 契约 |
| --- | --- |
| `text` | 必需的 `VARCHAR` expression，包含要生成 embedding 的文本。 |
| `options` | 可选的 foldable constant，通常是用 `struct_pack` 构造的 constant `STRUCT`。可以选择 `provider`、`model`、actor/batch/retry 控制、`dimensions`、`normalize`、request option 和 provider-specific 配置。 |

**返回值**

`dimensions` 缺失或为 `None` 时返回 `FLOAT[]`；`dimensions=N` 时返回 fixed-size `FLOAT[N]`。

**行为与数据契约**

该调用是保持行数的 Expression。Options 只绑定一次并转换为 Python 配置 dictionary，不能逐行变化；已识别的 provider 和 embedding 配置会被转发。`normalize=true` 请求 L2 normalization。输出可以在同一 projection 中与 ID 和源列组合。

**最小示例**

```python
import vane

con = vane.connect()
con.execute(
    """
    CREATE TEMP TABLE embedding_source AS
    SELECT *
    FROM (VALUES
        (1, 'invoice exception'),
        (2, 'delivery exception')
    ) AS source(document_id, text)
    """
)

result = con.sql(
    """
    SELECT
        document_id,
        text,
        ai_embed(
            text,
            struct_pack(
                provider := 'openai',
                model := 'text-embedding-3-small',
                dimensions := 512,
                normalize := true
            )
        ) AS embedding
    FROM embedding_source
    """
)
```

**限制与错误**

第一个参数必须绑定为 `VARCHAR`；第二个参数必须可 fold。该函数只能作为 `SELECT projection` 中的顶层值，不能用在 predicate、grouping、join、aggregate argument 或 short-circuit expression 中。SQL AI 会递归拒绝类似凭据的字段，包括嵌套 key 和 JSON option bridge。凭据必须通过 `OPENAI_API_KEY` 等 worker 环境变量提供，绝不能放入 SQL options。Provider 必须支持文本 embedding，并且请求的维度必须对该 provider 和 model 有效。

### Python 入口

Python builder 构建相同的单列 Expression 模型，适用于在 Python 中组装 projection。

#### `vane.ai.prompt`

**用途**

构建一个 lazy、保持行数的 Expression，为每个输入 message 生成文本响应。

**签名**

```python
vane.ai.prompt(messages, *, provider="openai", model=None, provider_options=None, prompt_options=None, system_message=None) -> Expression
```

**参数**

| 参数 | 契约 |
| --- | --- |
| `messages` | Vane/DuckDB `Expression` 或 literal message 值。 |
| `provider` | Provider 名称或 Provider 对象；默认为 `"openai"`。 |
| `model` | Provider model 标识符；`None` 使用 provider 默认值。 |
| `provider_options` | Typed provider option 或 provider-specific 扩展 dictionary。 |
| `prompt_options` | Typed request option 或 provider-specific 扩展 dictionary。 |
| `system_message` | 可选 system instruction；默认为 `None`。 |

**返回值**

返回输出类型为 `VARCHAR` 的 lazy `Expression`。用 `.alias(...)` 选择 projection 列名。

**行为与数据契约**

该 expression 增加一个保持行数的 projection 列。它会序列化轻量 provider descriptor，provider 在 worker 上延迟实例化。Provider concurrency 未设置时使用一个 actor，active runner 在规划时将它解析为 local subprocess 或 Ray backend。

**最小示例**

```python
import vane

con = vane.connect()
rel = con.sql(
    "SELECT 1 AS document_id, "
    "'Summarize the invoice exception.'::VARCHAR AS messages"
)

result = rel.select(
    vane.col("document_id"),
    vane.col("messages"),
    vane.ai.prompt(
        vane.col("messages"),
        provider="openai",
        model="gpt-4o-mini",
        system_message="Return one concise sentence.",
    ).alias("response"),
)
```

**限制与错误**

Expression 模式会拒绝 Relation-only 的 `return_format`、`image_columns`、`output_column`、`use_chat_completions` 和 `execution_backend` 参数；这些能力应使用 `rel.prompt`。该 expression 只支持 projection 位置，所选 provider 必须实现 prompting。相比在 Python 配置中放入 API key，应优先使用 worker 凭据环境变量。

#### `vane.ai.embed`

**用途**

构建一个 lazy、保持行数的 Expression，为每个输入行的文本生成 embedding。

**签名**

```python
vane.ai.embed(text, *, provider="openai", model=None, provider_options=None, embedding_options=None, dimensions=None, normalize=None) -> Expression
```

**参数**

| 参数 | 契约 |
| --- | --- |
| `text` | Vane/DuckDB `Expression` 或 literal text 值。 |
| `provider` | Provider 名称或 Provider 对象；默认为 `"openai"`。 |
| `model` | Provider model 标识符；`None` 使用 provider 默认值。 |
| `provider_options` | `OpenAIProviderOptions`、`GoogleProviderOptions` 或 provider-specific 扩展 dictionary。 |
| `embedding_options` | `OpenAIEmbeddingOptions`、`GoogleEmbeddingOptions` 或 request 扩展 dictionary。 |
| `dimensions` | `None` 表示动态输出，整数 `N` 表示 fixed-size vector。 |
| `normalize` | `True` 请求 L2 normalization；`None` 和 `False` 保留 provider 的未归一化输出。 |

**返回值**

返回 lazy `Expression`：`dimensions=None` 时为 `FLOAT[]`，`dimensions=N` 时为 `FLOAT[N]`。用 `.alias(...)` 选择输出名。

**行为与数据契约**

该 expression 增加一个保持行数的 projection 列。它会序列化轻量 provider descriptor，并在 worker 上延迟实例化 provider。Provider concurrency 未设置时使用一个 actor，active runner 在 query 规划时解析 execution backend。

**最小示例**

```python
import vane

con = vane.connect()
rel = con.sql(
    "SELECT 1 AS document_id, 'invoice exception'::VARCHAR AS text"
)

result = rel.select(
    vane.col("document_id"),
    vane.col("text"),
    vane.ai.embed(
        vane.col("text"),
        provider="openai",
        model="text-embedding-3-small",
        dimensions=512,
        normalize=True,
    ).alias("embedding"),
)
```

**限制与错误**

该 expression 只支持 projection 位置。Provider 必须实现文本 embedding，并且请求的维度必须对其 model 有效。相比在 Python 配置中放入 API key，应优先使用 worker 凭据环境变量。

## Relation API

这些 Python Relation method 是专用的表形转换。它们支持 Expression 入口没有的控制，也没有对应的 SQL table-function 语法。

#### `rel.prompt`

**用途**

以专用 Relation 转换的形式生成文本、多模态或结构化 prompt 输出。

**签名**

```python
rel.prompt(column, *, image_columns=None, provider="openai", model=None, provider_options=None, prompt_options=None, system_message=None, return_format=None, use_chat_completions=True, output_column="response", execution_backend=None, **options) -> Relation
```

**参数**

| 参数 | 契约 |
| --- | --- |
| `column` | 必需的文本/message 输入列名。 |
| `image_columns` | 用于多模态 prompt 的图像列名；默认为 `None`。 |
| `provider` | Provider 名称或对象；默认为 `"openai"`。 |
| `model` | Provider model 标识符；`None` 使用 provider 默认值。 |
| `provider_options` | Typed provider option 或 dictionary。 |
| `prompt_options` | Typed prompt option 或 dictionary。 |
| `system_message` | 可选 system instruction。 |
| `return_format` | Provider 支持的可选结构化 response model 或 schema。 |
| `use_chat_completions` | 选择 OpenAI-compatible chat 路径；默认为 `True`。 |
| `output_column` | 结果列名；默认为 `"response"`。 |
| `execution_backend` | 显式 Relation UDF backend；`None` 表示从 active runner 推断。 |
| `**options` | 其他 provider-specific 扩展。 |

**返回值**

返回一个 `Relation`，其配置的 `output_column` 为 `VARCHAR`；结构化结果会序列化到该列。

**行为与数据契约**

该 helper 读取 `column` 和任何 `image_columns`，批量执行 provider 调用，并返回配置的结果列。`return_format` 启用 provider 支持的结构化输出。与 Expression projection 不同，该 Relation 转换不承诺保留每个源列；如需把结果 join 回去，应在外围 workflow 中保留稳定 key。

**最小示例**

```python
import vane
from pydantic import BaseModel


class Decision(BaseModel):
    approved: bool
    reason: str


con = vane.connect()
rel = con.sql(
    "SELECT 1 AS document_id, "
    "'Approve the invoice exception.'::VARCHAR AS question, "
    "NULL::BLOB AS image"
)

result = rel.prompt(
    "question",
    image_columns=["image"],
    provider="openai",
    return_format=Decision,
    use_chat_completions=True,
    output_column="decision",
    execution_backend="subprocess_actor",
)
```

**限制与错误**

`column` 和每个指定的图像列都必须存在。Provider/model 必须支持请求的文本、图像和结构化输出能力。无效 `return_format`、不受支持的 chat 模式、缺少 worker dependency 或凭据，以及无效 backend 选择会在执行前或执行中失败。Provider 调用是可能重试的外部副作用。

[结构化与多模态 prompt 任务](/zh-CN/docs/data/guides/ai-functions) · [这些能力为何使用 Relation API](/zh-CN/docs/data/concepts/ai-functions)

#### `rel.embed_text`

**用途**

对 Relation 文本列生成 embedding，并可选使用内置字符分块。

**签名**

```python
rel.embed_text(column, *, provider=None, model=None, dimensions=None, output_column="embedding", max_chunk_chars=None, chunk_overlap_chars=200, execution_backend=None, **options) -> Relation
```

**参数**

| 参数 | 契约 |
| --- | --- |
| `column` | 必需的文本输入列名。 |
| `provider` | Provider 名称或对象；`None` 选择 `"transformers"`。 |
| `model` | Provider model 标识符；`None` 使用 provider 默认值。 |
| `dimensions` | 请求的 provider 维度；`None` 使用其默认值。 |
| `output_column` | 结果列名；默认为 `"embedding"`。 |
| `max_chunk_chars` | 每个 chunk 的最大字符数；`None` 禁用分块。 |
| `chunk_overlap_chars` | 相邻 chunk 的 overlap；默认为 `200`，仅在启用分块时使用。 |
| `execution_backend` | 显式 Relation UDF backend；`None` 表示从 active runner 推断。 |
| `**options` | 其他 provider-specific 扩展。 |

**返回值**

返回一个 `Relation`，其配置的输出列具有动态 `FLOAT[]` 类型。

**行为与数据契约**

默认 provider 是 `transformers`。设置 `max_chunk_chars` 时，文本会被分成带 overlap 的字符 chunk；chunk embedding 按文本长度加权平均后归一化。Relation helper 返回配置的输出列，不声明 fixed-size `FLOAT[N]`。

**最小示例**

```python
import vane

con = vane.connect()
rel = con.sql(
    "SELECT 1 AS document_id, "
    "'A long policy paragraph to embed.'::VARCHAR AS text"
)

result = rel.embed_text(
    "text",
    provider="transformers",
    model="sentence-transformers/all-MiniLM-L6-v2",
    max_chunk_chars=1000,
    chunk_overlap_chars=200,
    output_column="embedding",
    execution_backend="subprocess_actor",
)
```

**限制与错误**

`column` 必须存在并包含兼容的文本。Chunk size 与 overlap 必须构成有效配置，provider 必须实现文本 embedding，选定 backend 必须具有 provider dependency 和 model。需要保持行数的 projection 或可选 fixed-size `FLOAT[N]` 声明时，应使用 `vane.ai.embed` 或 `ai_embed`。

#### `rel.classify_text`

**用途**

使用必需的 zero-shot label 集合对每个文本值分类。

**签名**

```python
rel.classify_text(column, *, labels, provider=None, model=None, output_column="label", execution_backend=None, **options) -> Relation
```

**参数**

| 参数 | 契约 |
| --- | --- |
| `column` | 必需的文本输入列名。 |
| `labels` | 必需的候选 label 字符串列表。 |
| `provider` | Provider 名称或对象；`None` 选择 `"transformers"`。 |
| `model` | Provider model 标识符；`None` 使用 provider 默认值。 |
| `output_column` | 结果列名；默认为 `"label"`。 |
| `execution_backend` | 显式 Relation UDF backend；`None` 表示从 active runner 推断。 |
| `**options` | 其他 provider-specific 扩展。 |

**返回值**

返回一个 `Relation`，其配置的输出列具有 `VARCHAR` 类型。

**行为与数据契约**

默认 provider 是 `transformers`。Classifier 针对提供的 label 为每个输入评分，并在 `output_column` 中返回选中的 label。

**最小示例**

```python
import vane

con = vane.connect()
rel = con.sql(
    "SELECT 1 AS document_id, "
    "'The invoice total does not match.'::VARCHAR AS text"
)

result = rel.classify_text(
    "text",
    labels=["billing", "delivery", "other"],
    provider="transformers",
    model="facebook/bart-large-mnli",
    output_column="label",
    execution_backend="subprocess_actor",
)
```

**限制与错误**

`column` 必须存在，`labels` 是必需参数，所选 provider 必须实现文本分类。Backend worker 必须具有 provider dependency 和 model。在当前 API 版本中，分类没有 SQL 或 Python Expression 等价接口。

## AI 共享类型与限制

下列 option class 是 frozen public dataclass。其公开字段和默认值为精确契约；在签名允许时，dictionary 和 provider-specific 扩展仍然受支持。

| 类型 | 公开字段和默认值 |
| --- | --- |
| `OpenAIProviderOptions` | `base_url=None, api_key=None, organization=None, timeout=None, concurrency=None, max_api_concurrency=None` |
| `OpenAIPromptOptions` | `use_chat_completions=None, max_output_tokens=None, max_tokens=None, temperature=None, on_error=None` |
| `OpenAIEmbeddingOptions` | `encoding_format="float", on_error=None` |
| `AnthropicProviderOptions` | `api_key=None, base_url=None, timeout=None, max_retries=None, concurrency=None, max_api_concurrency=None` |
| `AnthropicPromptOptions` | `max_tokens=None, temperature=None, top_p=None, top_k=None, stop_sequences=None, on_error=None` |
| `GoogleProviderOptions` | `api_key=None, concurrency=None, max_api_concurrency=None` |
| `GooglePromptOptions` | `max_output_tokens=None, temperature=None, top_p=None, top_k=None, on_error=None` |
| `GoogleEmbeddingOptions` | `task_type=None, title=None, on_error=None` |
| `VLLMProviderOptions` | `engine_args=None, concurrency=None, gpus_per_actor=None` |
| `VLLMPromptOptions` | `generate_args=None, max_tokens=None, temperature=None, on_error=None` |

`concurrency` 映射为 AI actor 数。`max_api_concurrency` 独立限制每个 actor 内同时处理的 provider request。

### Provider 默认值

| Provider 能力 | 默认 model 或资源 |
| --- | --- |
| OpenAI embedding | `text-embedding-3-small`；provider batch `64`；API batch token 上限 `300000` |
| OpenAI prompt | `gpt-4o-mini`；API 最大并发度 `32` |
| Anthropic prompt | `claude-sonnet-4-20250514`；API 最大并发度 `16` |
| Google embedding | `text-embedding-004`；未知 model 维度 fallback `768` |
| Google prompt | `gemini-2.0-flash`；API 最大并发度 `16` |
| vLLM prompt | `Qwen/Qwen3-1.7B`；`gpus_per_actor=1`；wrapper retry `0` |
| Transformers embedding | `sentence-transformers/all-MiniLM-L6-v2`；batch `64`；检测到 CUDA 时 GPU request `1` |
| Transformers classification | `facebook/bart-large-mnli` |

### Provider 生命周期与执行限制

- 每个 query 都携带轻量 provider descriptor。真实 client 或 model 在 worker 上延迟实例化，并在该 actor 的后续 batch 中复用。
- Provider option 未覆盖时，Expression 和 SQL AI 默认使用一个 actor、请求零个 GPU，batch size 为 `32`。Relation prompt fallback batch size 为 `1`。
- active runner 把 Expression 和 SQL AI 解析为 local subprocess actor 或 Ray actor。Relation method 还可以显式设置 `execution_backend`。
- Ray GPU 资源请求会通过 Ray scheduler 预留资源。local subprocess 只继承 driver 可见的 GPU：它不提供 GPU reservation 或隔离，非零 GPU 请求会产生 warning。
- 每个 worker 都需要 provider dependency、model file、endpoint 访问能力和凭据环境变量。虽然 Python provider option class 公开了凭据字段，仍应优先使用 worker 环境配置；SQL options 会递归拒绝类似凭据的字段。
- Expression 和 SQL 调用只能位于顶层 `SELECT projection`；不支持 `WHERE`、`GROUP BY`、`HAVING`、join、aggregate argument、`CASE`、`AND`、`OR` 和 `COALESCE`。
- Provider 和 wrapper retry 可能重复外部调用。Exponential retry sleep 上限为 30 秒，`Retry-After` sleep 上限为 120 秒；外部副作用不具有事务性，也不提供 exactly-once。应保留稳定 ID，并使下游写出具有幂等性或可审查性。
- 导入 `vane` 时会动态挂载 Relation AI method。当前尚无完整的 public type-stub 覆盖，因此即使这些 method 在运行时是公开接口，static type checker 仍可能需要本地注解。

配置和完整 workflow 请参阅 [AI Function 指南](/zh-CN/docs/data/guides/ai-functions)。Provider 生命周期、位置和失败边界的设计原因请参阅 [AI Function 概念](/zh-CN/docs/data/concepts/ai-functions)。
